    2 色彩科學

    光、眼睛與數位影像

    工具可以學，原理一旦通，工具會自己說話——這一章決定你的調色上限

    人眼如何感知色彩
    三錐細胞與色彩感知

    人眼視網膜上有三種錐細胞，分別對不同波長敏感：

    1
    L 錐（長波長）— 對紅橙色敏感，峰值約 564nm
    2
    M 錐（中波長）— 對綠色敏感，峰值約 534nm
    3
    S 錐（短波長）— 對藍紫色敏感，峰值約 420nm
    數位攝影機模擬的是人眼，不是物理現實

    攝影機的 RGB 三通道，本質上是在模擬 L、M、S 三種錐細胞的反應。

    這解釋了為什麼兩台攝影機拍同一個場景，色彩會不同——它們對光的「理解」不同，就像兩個人對同一首音樂的感受不一樣。

    色彩的三個維度

    任何色彩都可以用三個維度描述，調色工具的設計都圍繞這三個維度：

    色相（Hue）：顏色的「種類」，紅/橙/黃/綠/藍/紫，圍成一個圓 飽和度（Saturation）：顏色的純度，從灰色（0%）到純色（100%） 亮度（Lightness / Luminance）：顏色的明暗程度，從黑到白
    色彩名詞解釋
    色域（Color Gamut）
    一個色彩系統能表現的全部色彩範圍，色域越大能顯示的颜色越多
    色溫（Color Temperature）
    以克耳文（K）為單位的光源顏色指標，數值越低越偏藍（冷），越高越偏橙（暖）
    LUT（Look-Up Table）
    色彩查找表，將輸入色彩值映射到輸出值，用於快速套用風格或技術轉換
    Gamma
    亮度的非線性編碼曲線，決定數值與實際感知亮度的對應關係
    動態範圍（Dynamic Range）
    影像能同時記錄的最亮與最暗區域的比值，單位為 stops
    色彩空間（Color Space）
    定義色彩如何數位化編碼與解碼的完整規範，包含色域與 Gamma
    廣播安全（Broadcast Safe）
    亮度與色度不超出電視廣播標準的範圍，確保在所有接收設備上正確顯示
    色彩視覺的三個恆常現象
    1
    色彩恆常性
    同一物體在不同光線下，大腦會自動補正為「正確」顏色（白紙在黃光下仍被感知為白色）
    2
    同時對比效應
    同一灰色在深色背景上顯得淺，在淺色背景上顯得深
    3
    色彩疲勞效應
    長時間看某種顏色後，感知會偏移到補色方向
    「調色疲勞」是調色師最危險的職業病

    長時間盯著同一畫面調色，眼睛會對當前色調適應，導致過度校正。

    專業調色師的習慣：每 30-45 分鐘休息 5 分鐘，或用「中性灰眼神飛秒測試」——閉眼 30 秒後快速看畫面，感受第一眼的色偏印象。

    • 每隔 30 分鐘起身走動，讓眼睛遠離螢幕
    • 準備一張標準色卡紙放在旁邊，偶爾看一眼作為「色彩重置」
    • 不要在疲勞時做最終確認——隔天早上再看往往會發現問題
    • 使用 DaVinci 的「A/B 比較」功能，切換回原始畫面快速重置視覺
    數位影像的色彩模型
    RGB 色彩空間
    text RGB 色彩模型的實際運作 
    每個像素有 R（紅）、G（綠）、B（藍）三個通道值

8-bit：  每通道 256 階（0-255），共 16.7 百萬色
10-bit：每通道 1024 階，共 10.7 億色
12-bit：每通道 4096 階，共 680 億色
16-bit：每通道 65536 階，用於 HDR 後製

調色時的意義：
  R=255, G=255, B=255 → 純白
  R=0,   G=0,   B=0   → 純黑
  R=255, G=0,   B=0   → 純紅
  R=255, G=165, B=0   → 橙色
    色域（Color Gamut）與色彩空間

    色域決定了一個系統能表現的色彩範圍：

    sRGB / Rec.709：標準螢幕色域，涵蓋可見光的約 35%，適合網頁與一般影片 DCI-P3：電影院標準，約為 sRGB 的 1.5 倍，Apple 顯示器與高階手機使用 Rec.2020：HDR 標準，涵蓋可見光的約 75%，廣播電視 HDR 標準 ACES：學院色彩編碼系統，完整涵蓋所有攝影機可拍攝的色彩範圍
    Gamma 曲線與感知亮度

    人眼對亮度的感知是非線性的——對暗部變化更敏感，對亮部變化相對不敏感：

    text Gamma 曲線的本質 
    線性光（Linear Light）：攝影機感光元件的真實反應，光量翻倍 = 數值翻倍
人眼感知（Perceptual）：非線性，暗部辨識力更高，需要更多 bit 保存暗部細節

Gamma 2.2（sRGB）：電腦螢幕標準
Gamma 2.4（Rec.709）：電視廣播標準
LOG Gamma：高反差壓縮，保留最大動態範圍用於後製
    LOG 格式：專業攝影機的秘密
    為什麼要用 LOG 拍攝

    LOG（對數）格式是攝影機廠商設計的一種特殊 Gamma 曲線，目的是在影像感光器有限的 bit depth 內，盡可能保留最大的動態範圍：

    1
    原始場景可能有 14-16 stops 的動態範圍（高光到陰影的亮度比）
    2
    標準 Rec.709 只能記錄約 6-8 stops
    3
    LOG 用對數壓縮，把 14 stops 的資訊塞進 10-12 bit 的影像檔案
    4
    代價是：直接看起來像「洗白」「低對比」「灰矇矇」
    5
    調色時還原 Gamma，再做風格化處理
    LOG 與 Rec.709 視覺對比
    LOG 原始畫面
    套用 Rec.709 轉換後
    低對比，畫面灰矇矇
    對比正常，暗部紮實高光通透
    色彩平淡，飽和度極低
    色彩飽滿鮮明
    保留 14+ stops 動態範圍
    約 6-8 stops 可用範圍
    不適合直接觀看或交付
    適合直接輸出到 YouTube / 電視
    必須經過色彩轉換才能調色
    可直接進行風格化調色
    各廠商 LOG 格式對照
    Sony S-Log2 / S-Log3：Sony Alpha 與攝影機系列，S-Log3 動態範圍更大 Panasonic V-Log / V-Log L：GH5/GH6/S5II 系列，V-Log L 為 V-Log 的輕量版 Canon Log 2 / Log 3：Cinema EOS 系列，C70/C300 Mark III DJI D-Log / D-Log M：大疆無人機與運動相機，M 版保留更多亮部
    text 在 DaVinci 正確處理 LOG 素材 
    方法一：Color Space Transform（CST）節點
→ 節點庫搜尋 CST → 設定輸入色彩空間（如 S-Gamut3.Cine / S-Log3）
→ 設定輸出色彩空間（Rec.709 / Gamma 2.4）
→ 此節點放在節點樹最前端

方法二：在 Color 管理中設定（推薦新手）
→ 專案設定 → 色彩管理 → 輸入色彩空間設為攝影機 LOG
→ 時間軸色彩空間設為 Rec.709
→ DaVinci 自動轉換所有素材

方法三：套用官方提供的技術 LUT
→ 各廠商提供 LOG → Rec.709 的 LUT 檔案（可官網免費下載）
→ 在調色節點最前端套用此 LUT

    第一次接觸 LOG 素材時，建議先用「方法二」在 Color 管理中設定自動轉換。這樣你不需要在每個鏡頭手動處理技術轉換，可以專注在創意調色上。等熟悉後再改用 CST 節點獲得更精確的控制。

    Waveform（波形圖）

    波形圖是最基礎也最重要的示波器。橫軸對應畫面的左右位置，縱軸是亮度值（0-100%）：

    text 波形圖解讀指南 
    讀取方式：把波形圖想成把畫面「側過來看」

0%（底部）   = 畫面中最暗的區域（理論純黑）
100%（頂部） = 畫面中最亮的區域（理論純白）

廣播安全範圍：7.5% - 100%（Rec.709 廣播標準）
一般螢幕範圍：0% - 100%

常見問題：
  波形集中在底部 → 整體曝光不足，需要提亮
  波形超過 100%   → 過曝，高光細節丟失（無法救回）
  波形只在 20-70% → 對比度不足，畫面「平」
    「死白」與「死黑」

    當波形觸及 0% 下方（死黑）或 100% 以上（死白），代表該區域像素完全沒有細節。

    死白是拍攝時的問題（曝光過度），調色無法救回。死黑則在調色時可以適當利用——電影感調色往往刻意保留一點「純黑」區域增加深度感。

    一旦拍攝時高光過曝（死白），該區域的像素資訊已經完全丟失，調色軟體無能為力。這就像在錄音時爆音一樣，後製無法恢復原本的聲音。所以拍攝時寧可欠曝一點（往左偏），也不要過曝——欠曝的暗部至少還保有可以拉回的資訊。

    Parade（分量圖）

    Parade 圖同時顯示 RGB 三個通道的獨立波形，是判斷色偏的最直接工具：

    1
    三條線等高 → 中性色，無色偏（白色物體應呈現此狀態）
    2
    R 通道偏高 → 整體偏紅（暖色偏）
    3
    B 通道偏高 → 整體偏藍（冷色偏）
    4
    R 高 G 低 B 低 → 偏紅
    5
    R 低 G 低 B 高 → 偏藍
    6
    R 低 G 高 B 低 → 偏綠
    text 用 Parade 圖判斷色偏範例 
    場景：在鎢絲燈光下拍攝的訪談

看到 Parade 圖：R 通道比 G、B 高 10-15%
→ 整體偏暖橘色（鎢絲燈白平衡未校正）
→ 解法：在 Offset（全域）中往青藍方向微調
→ 調整到 RGB 三條線等高，白色物體變中性白
    Vectorscope（向量圖）

    向量圖顯示色相（Hue）和飽和度（Saturation）的分布，呈圓形：

    圓心位置 = 中性灰/白/黑（無色彩） 距圓心距離 = 飽和度（越遠越飽和） 角度方向 = 色相（圓圈上的 R/G/B/Cy/Mg/Yl 六個目標點） 膚色線（Skin Tone Line）= 從圓心到右側約 10-11 點鐘方向的參考線，健康膚色應落在此線附近
    Histogram（直方圖）與 CIE 色度圖
    text Histogram 讀取要點 
    橫軸：亮度（0=黑，255 或 1023=白）
縱軸：該亮度的像素數量

理想人像畫面：
  鐘形分布，中間調最多像素
  暗部（左側）略有分布代表陰影細節存在
  高光（右側）輕微分布代表適當高光

過曝跡象：直方圖右側被截斷（有很高的牆）
欠曝跡象：直方圖左側被截斷

CIE 色度圖（Chromaticity Diagram）：
  顯示色彩的絕對色度位置，用於判斷是否超出目標色域
  在 HDR 與廣播工作中重要
    4 節點系統

    DaVinci 調色的核心架構

    節點是 DaVinci 最強的設計，也是大多數人最先放棄的地方——理解它，一切迎刃而解

    節點的基本概念
    圖層思維 vs 節點思維
    Photoshop 圖層思維
    DaVinci 節點思維
    上方圖層蓋住下方圖層
    像水管，資訊從左流向右
    疊加方向從下到上
    每個節點做一件事
    適合靜態影像處理
    可做分支、混合、非線性處理

    讓畫面技術上正確

    調色分兩個階段：先讓畫面「正確」，再讓它「好看」——這章解決「正確」的問題

    系統化一級調色流程
    五步驟技術校正
    1
    LOG 轉換 → 確認節點 01 正確套用了 CST 或技術 LUT，素材看起來不再灰矇矇
    2
    曝光評估 → 看波形圖，確認主體亮度在 50-70%，高光不超過 100%
    3
    色偏校正 → 看 Parade 圖，調整讓中性色（白牆、灰卡）的 RGB 三線等高
    4
    對比建立 → 拉 S 形曲線，讓暗部紮實、高光有層次
    5
    最終確認 → 再次看波形、Parade、Vectorscope 確認所有數值在安全範圍
    新手最常見的順序錯誤

    很多新手會先調整對比或風格，結果發現怎麼調都不對。原因是：在素材還未達到技術標準前做任何風格化調整，都是在沙地上蓋房子。

    正確的順序永遠是：技術校正（曝光、白平衡、對比）→ 風格創作（LUT、色彩偏移）。前者讓畫面「正確」，後者讓畫面「好看」。兩者順序顛倒會事倍功半。

    Lift / Gamma / Gain / Offset 的定義

    對比度控制技術
    什麼是「好的對比度」

    對比度不是越高越好，而是要符合內容的情緒需求：

    1
    高對比度（Contrast > 1.0，Pivot 0.5）— 戲劇性、衝突感、恐怖/驚悚類型
    2
    中對比度（Contrast 0.9-1.1）— 一般紀錄片、訪談、真實感
    3
    低對比度（Contrast < 0.9，Lift 提亮）— 夢幻、懷舊、日系風格
    4
    膠片對比度（S 形曲線，Lifted Blacks）— 電影感，暗部不完全死黑
    text 精確對比度控制的四種方法 
    方法一：Contrast + Pivot 滑桿
  Contrast：整體對比倍率
  Pivot：對比變化的中心點（設為 0.5 = 以中間灰為基準）

方法二：Lift + Gain 分別控制
  降低 Lift = 壓深陰影
  提高 Gain = 拉亮高光
  → 同時做 = 增加對比，適合精細控制兩端

    精細控制每一個色彩維度

    Curves 是調色師的手術刀——Color Wheels 是斧頭，Curves 是解剖刀

    Custom Curves（自訂曲線）
    曲線的基本邏輯

    曲線圖橫軸是輸入值（原始畫面），縱軸是輸出值（調色後畫面）。對角線是「未調整」狀態：

    1
    把曲線往左上拖 → 提亮（同樣的輸入，輸出更亮）
    2
    把曲線往右下拖 → 壓暗（同樣的輸入，輸出更暗）
    3
    在暗部往上 + 高光往下 → S 形曲線，增加對比
    4
    在暗部往上（提亮暗部）→「Lifted Blacks」電影感
    5
    在曲線底部打一個固定點，在高光拉高 → 只提亮高光，不動暗部
    S 形曲線的電影感應用
    text 電影感 S 形曲線具體操作 
    步驟 1：在暗部（約 25% 位置）加一個控制點，往下拉 5-8%
         → 讓陰影更有深度感

步驟 2：在高光（約 75% 位置）加一個控制點，往上拉 3-5%
         → 讓高光更通透

步驟 3：在最暗處（0% 附近）加一個固定點，往上提 2-5%
         → Lifted Blacks：讓陰影不完全死黑，產生膠片感

步驟 4：在最亮處（100% 附近）加一個限制點，輕微往下
         → 壓制高光，防止過曝同時增加電影感的高光壓制

注意事項：
  控制點不超過 4-5 個，太多控制點曲線會震盪
  使用「平滑（Smooth）」選項讓曲線過渡更自然

    過度拉 S 形曲線是新手最常犯的錯誤——暗部拉太深導致死黑，高光拉太高導致死白。每次調整後務必看波形圖確認：暗部不低於 2%，高光不超過 98%。好的 S 形曲線是讓畫面「有對比」而不是「有死黑死白」。

    分通道曲線（YRGB Curves）

    DaVinci 的曲線允許獨立控制 Y（亮度）、R、G、B 四個通道：

    Y 通道（亮度曲線）：只調整亮度，不影響色相與飽和度，最乾淨的亮度控制 R 通道：往上提亮紅色 / 往下壓暗青色（紅的補色） G 通道：往上提亮綠色 / 往下壓暗洋紅 B 通道：往上提亮藍色 / 往下壓暗橙色（金黃色）
    text 分通道曲線的創意應用 
    創造暖色高光 + 冷色陰影（電影常見手法）：
  R 通道：在高光位置（75%）微微往上，在暗部（25%）微微往下
  B 通道：在暗部（25%）微微往上，在高光（75%）微微往下
  → 結果：高光偏暖橙，陰影偏冷藍（即 Orange & Teal 的基礎）

創造綠色陰影（恐怖、科技感）：
  G 通道：在暗部（15%）往上拉
  → 讓陰影帶有綠色調，搭配冷色整體色調

創造夕陽感高光：
  R 通道：高光（80%+）大幅往上
  B 通道：高光（80%+）往下壓
  → 高光呈現橙紅色，像夕陽光照效果

    Hue vs. Hue（色相轉移）

    輸入色相→輸出色相的轉換，可以把某個顏色「推向」另一個方向：

    text Hue vs. Hue 實際應用 
    場景：秋天拍攝，希望樹葉更偏金黃（原本偏綠黃）

步驟：
1. 在 Hue vs. Hue 曲線找到黃綠色的位置
2. 用吸管在監視器點取要調整的顏色（自動定位到該色相）
3. 把控制點往橙黃方向拖（往右 = 往橙紅方向偏移）

常見應用：
  天空偏青 → 往藍色方向推（更乾淨的天空藍）
  草地偏黃 → 往綠色方向推（更鮮活的草綠）
  膚色偏橙 → 微微往黃方向拉（更自然的東方膚色）
    Hue vs. Saturation（選色飽和度）

    針對特定色相的飽和度控制，不影響其他顏色：

    text Hue vs. Saturation 常見用法 
    降低整體飽和度但保留膚色：
  1. 整體：Saturation 滑桿降至 70-80%（整體去飽和）
  2. Hue vs. Sat：找到膚色（橙黃）位置，把控制點往上拉
  → 結果：背景色彩淡雅，但主體膚色仍然飽滿

強調天空的藍色：
  找到天空藍色位置 → 往上拉

    8 膚色處理

    人物調色的關鍵技術

    觀眾對膚色最敏感——膚色不對，整個畫面的真實感就崩塌了

    膚色的生理基礎
    膚色為什麼難處理

    人類皮膚的色彩構成非常複雜：

    1
    黑色素（Melanin）— 決定膚色深淺，從淺粉到深褐
    2
    血紅素（Hemoglobin）— 血管分布帶來的紅色，在臉頰、嘴唇、耳朵特別明顯
    3
    光線散射（Subsurface Scattering）— 光線進入皮膚後散射，讓皮膚看起來「通透」而非不透明
    4
    環境反射
    周圍環境顏色反射在皮膚表面，是最常見的膚色問題來源
    向量圖與膚色線
    text 用向量圖判斷膚色品質 
    膚色線（Skin Tone Line）：
  從向量圖圓心，延伸到 11 點鐘方向（約 10-12 點方向）的一條參考線
  所有健康的人類膚色（無論深淺）都應該落在這條線附近

膚色正常：向量圖中膚色的點群落在膚色線附近
偏紅（發炎感）：點群偏向膚色線右側（往紅色方向）
偏黃（蠟黃感）：點群偏向膚色線左側（往黃色方向）
偏洋紅：點群往上偏移
偏橙：點群往右下偏移

判斷步驟：

2. 看向量圖中點群的分布位置
3. 判斷需要往哪個方向調整

    用 Qualifier 精確選取膚色

  S（飽和度）：避免選到白牆（低飽和），設在 15-80%
  L（亮度）：避免選到陰影死黑，設在 15-85%
    不同膚色的調整策略
    text 膚色調整的方向性建議 
    淺膚色（歐美白人）：
  常見問題：在特定光線下顯得過粉或過紅
  調整方向：Hue vs. Hue 把偏紅的膚色往橙黃方向微推 5-10°
  飽和度：Hue vs. Sat 降低膚色飽和度 10-15%，防止「紅臉」感

  常見問題：黃色成分過重，顯得「蠟黃」
  調整方向：Hue vs. Hue 把偏黃的膚色往橙色方向微推
  亮度補正：Luminance vs. Luminance 在膚色亮度範圍微微提亮
  
深膚色（非裔、南亞）：
  常見問題：在錯誤曝光下喪失細節，或在補光不足時完全沒有紋理
  調整方向：謹慎提亮陰影，保留皮膚紋理；飽和度通常不需降低
  重要：Log Wheels 的 Shadow → 微微提亮，保留暗部細節

  Vectorscope 向量圖確認膚色點群落在膚色線上
  波形圖確認臉部亮度在 50-75%（特寫訪談的標準亮度）
  不要為了「讓膚色好看」而破壞整體場景的色彩一致性

    處理膚色時，先用 Vectorscope 確認方向（偏紅/偏黃），再用 Qualifier 只選膚色範圍微調。記住：觀眾對膚色的敏感度遠超過其他色彩——膚色正確了，整個畫面的真實感就會大幅提升。如果時間有限，膚色校正的投資報酬率最高。

    美化技術與自然感的平衡
    輕微提亮臉部（Gamma +0.08 到 +0.12）：讓主角從背景中脫穎而出 降低膚色飽和度（Hue vs. Sat -10 到 -15%）：去除「紅臉」的不健康感 暗部提亮（Lift +0.03）：減少眼窩、法令紋的強烈陰影 Color Warper 膚色微調：在色彩空間中把膚色精確推向最理想的方向
    「美化」的邊界在哪裡？

    調色的目的是讓人看起來「最好的自己」，而不是「不像自己」。

    一個好用的測試：把調色前後並排，請另一個人看，問他「這兩張照片是同一個人嗎？」如果答案是「不確定」，你的美化已經超過邊界。調色師負責皮膚的光澤與色彩，不負責移除真實的特徵。

    電影感調色的核心要素
    什麼讓調色有「電影感」

    電影感不是一個調色參數，而是幾個技術選擇的組合：

    1
    Lifted Blacks（暗部提亮）— 讓陰影從純黑提到深灰，模擬膠片的暗部特性
    2
    Compressed Highlights（高光壓制）— 讓高光從純白壓下來，防止高光「死白」
    3
    降低飽和度
    膠片時代的色彩飽和度比數位低，整體降至 70-85%
    4
    Warmth Bias（暖色傾向）— 輕微暖色整體，使畫面更「有機感」
    5
    粒狀感（Grain）— 模擬膠片顆粒，讓畫面不那麼「數位感」
    text 電影感基礎設定參數 
    一、Lifted Blacks：

Step 3：匯出 LUT
  Color 頁面選單 → 「Color 管理」→「產生 LUT」
  選擇 33 點 LUT（最常用）
  匯出格式：.cube（最通用的格式）

Step 4：在其他鏡頭測試 LUT
  新建節點 → 右鍵 → LUTs → 選取剛才的 .cube 檔
  測試不同類型素材（人像、風景、室內、夜景）
  確認 LUT 在各種場景下的效果穩定

Step 5：調整 LUT 強度
  在套用 LUT 的節點 → 右鍵 → Key → Opacity
  調整不透明度（50-80%）讓 LUT 效果更融合

注意事項：
  LUT 是在一級校正後套用，不是替代一級校正
  每次使用 LUT 後仍需做局部微調（膚色、問題高光等）
    10 色彩管理

    從攝影機到螢幕的完整鏈路

    色彩管理是讓你在「所見即所得」環境下工作的關鍵基礎設施

    DaVinci 的色彩管理模式
    三種色彩管理工作流程
    無管理（Unmanaged）：素材直接進入節點，你自己負責所有轉換，適合有經驗的調色師 DaVinci YRGB Color Managed：DaVinci 自動處理輸入輸出色彩空間轉換，推薦一般工作流 ACES（Academy Color Encoding System）：電影院級別的科學色彩管理，適合高端製作
    text DaVinci Color Managed 設定步驟 
    Step 1：專案設定 → 色彩管理
  色彩科學：DaVinci YRGB Color Managed

Step 2：設定輸入色彩空間
  Automatic（自動）：DaVinci 根據素材元數據自動偵測
  手動設定：若自動偵測錯誤，選取正確的攝影機格式

Step 3：設定時間軸色彩空間
  通常設為 DaVinci Wide Gamut / DaVinci Intermediate
  → 這是內部工作色彩空間，範圍最大

Step 4：設定輸出色彩空間
  Rec.709 Gamma 2.4（廣播電視、YouTube）
  DCI-P3（電影院投影）
  Rec.2020 ST2084 PQ（HDR 輸出）

效果：
  所有 LOG 素材自動轉換到可工作的色彩空間
  輸出時自動轉換到目標格式
  你只需要專注在創意調色，不需要手動做技術轉換
    ACES 工作流程入門
    ACES 的核心概念

    ACES（Academy Color Encoding System）是美國電影藝術與科學學院制定的開放式色彩管理系統：

    1
    輸入：所有攝影機素材通過 IDT（Input Device Transform）轉換到 ACES 色彩空間
    2
    工作：在 ACES 巨大的色彩空間內進行所有調色工作
    3
    輸出：通過 ODT（Output Device Transform）轉換到目標顯示裝置
    4
    優點：不同攝影機素材自動統一到同一個色彩空間，色彩一致性最好
    11 職業工作法

    效率、一致性與交付

    調色技術只是工具，職業工作法決定你能不能可持續地做好的調色

    場景間色彩匹配
    分鏡色彩一致性

    實際工作中，同一場景的不同鏡頭可能是在不同時間、不同光線條件下拍攝的，調色師需要讓它們看起來像在同一時間：

    text 場景色彩匹配工作流程 
    方法一：參考靜態畫面（Stills）比對
  Step 1：選取「基準鏡頭」（光線最好的那一顆）完成調色
  Step 2：右鍵 → 「截取靜態畫面（Grab Still）」
  Step 3：其他鏡頭調色時，把基準鏡頭的 Still 拉到比較視窗
  Step 4：根據 Still 調整直到兩個鏡頭的示波器數值接近

方法二：自動色彩匹配（Colour Match）
  同時選取需要匹配的鏡頭和目標鏡頭
  Color 頁面選單 → 「Colour Match」
  → DaVinci 自動計算需要的調整，作為初始校正基礎
  → 通常還需要手動微調，但節省大量時間

方法三：使用相同 PowerGrade（調色預設）
  把基準鏡頭的節點樹存成 PowerGrade（右鍵 Gallery → 新增 PowerGrade）
  其他鏡頭右鍵 → 套用 PowerGrade
  → 節點結構完全複製，再做針對性微調
    批次調色與 PowerGrade
    跨鏡頭效率工具
    text 批次調色操作方法 
    方法一：Group 節點（群組前/後節點）
  在 Clip 縮略圖右鍵 → 「加入到群組」→ 選取多個鏡頭加入同一群組
  Group Pre-Clip：在個別調色前加入的共用節點（如 LOG 轉換）
  Group Post-Clip：在個別調色後加入的共用節點（如輸出保護、整體風格）
  → 修改 Group 節點，所有群組內鏡頭同時更新

方法二：Remote Grades（遠端調色版本）
  一個鏡頭多次出現時（如 A/B Roll 同一素材用了多次）
  右鍵 Clip → Remote Grades → 啟用
  → 修改原始鏡頭的調色，所有使用該素材的地方同步更新

方法三：PowerGrade 庫管理
  把常用的節點樹存為 PowerGrade
  命名規範：[風格]-[適用素材]-[版本]
  例如：FilmLook-Sony-v2、SkinCorrect-Asian-A7IV
    調色交付標準
    輸出前的最終確認清單
    text 調色完成前的品質確認 
    技術面：
  □ 所有鏡頭的波形圖在廣播安全範圍（0-100%）
  □ Parade 圖確認中性物體（白牆、灰卡）白平衡正確
  □ 沒有任何鏡頭有未修正的嚴重色偏
  □ 場景間的色彩一致性（用 Stills 比對）
  □ 膚色在向量圖的膚色線附近

創意面：
  □ 整體調色風格符合導演/客戶的參考要求
  □ 情緒化場景的色彩強度符合敘事需要
  □ 夜景場景與日間場景的風格統一但有區別

輸出面：
  □ 確認輸出色彩空間（YouTube 用 Rec.709 / 電影用 DCI-P3）
  □ 確認是否需要輸出 LUT 檔案給客戶
  □ 交付 Still 庫（客戶要求修改時的參考基準）
    與剪輯師/導演的協作
    1
    取得調色參考（Reference）— 客戶或導演提供的「目標影片」，作為風格方向
    2
    建立調色意見交換流程
    可用 DaVinci 的「Timeline Comment」或共享 Stills
    3
    版本管理
    使用 DaVinci 的「版本（Version）」功能保存多個調色版本
    4
    遠端審片
    輸出代理檔（1080p H.264）給客戶審閱，確認後再做最終輸出
    進階測驗：綜合應用
    要建立「高光偏暖、陰影偏冷」的電影感效果，應該如何使用分通道曲線？
    往上是該通道顏色增強，往下是其補色增強。要高光偏暖（紅/橙）就拉 R 高光，要陰影偏冷（藍）就拉 B 暗部
    在節點樹中，創意風格 LUT 應該放在一級校正節點之後？
    LUT 是在畫面已經技術正確的前提下疊加風格，不是替代校正
    健康人類的膚色在 Vectorscope 上應該落在哪裡？
    膚色線從圓心延伸到約 11 點鐘方向，所有膚色（無論深淺）都應落在此線附近
