# 色彩科學：光、眼睛與數位影像
> 工具可以學，原理一旦通，工具會自己說話——這一章決定你的調色上限

## 人眼如何感知色彩

### 三錐細胞與色彩感知

人眼視網膜上有三種錐細胞，分別對不同波長敏感：

[flow]
1. L 錐（長波長）— 對紅橙色敏感，峰值約 564nm
2. M 錐（中波長）— 對綠色敏感，峰值約 534nm
3. S 錐（短波長）— 對藍紫色敏感，峰值約 420nm
[/flow]

> **數位攝影機模擬的是人眼，不是物理現實**
> 攝影機的 RGB 三通道，本質上是在模擬 L、M、S 三種錐細胞的反應。
>
> 這解釋了為什麼兩台攝影機拍同一個場景，色彩會不同——它們對光的「理解」不同，就像兩個人對同一首音樂的感受不一樣。

### 色彩的三個維度

任何色彩都可以用三個維度描述，調色工具的設計都圍繞這三個維度：

[tags]
- [blue] 色相（Hue）：顏色的「種類」，紅/橙/黃/綠/藍/紫，圍成一個圓
- [orange] 飽和度（Saturation）：顏色的純度，從灰色（0%）到純色（100%）
- [green] 亮度（Lightness / Luminance）：顏色的明暗程度，從黑到白
[/tags]

### 色彩名詞解釋

[dl]
- 色域（Color Gamut） | 一個色彩系統能表現的全部色彩範圍，色域越大能顯示的颜色越多
- 色溫（Color Temperature） | 以克耳文（K）為單位的光源顏色指標，數值越低越偏藍（冷），越高越偏橙（暖）
- LUT（Look-Up Table） | 色彩查找表，將輸入色彩值映射到輸出值，用於快速套用風格或技術轉換
- Gamma | 亮度的非線性編碼曲線，決定數值與實際感知亮度的對應關係
- 動態範圍（Dynamic Range） | 影像能同時記錄的最亮與最暗區域的比值，單位為 stops
- 色彩空間（Color Space） | 定義色彩如何數位化編碼與解碼的完整規範，包含色域與 Gamma
- 廣播安全（Broadcast Safe） | 亮度與色度不超出電視廣播標準的範圍，確保在所有接收設備上正確顯示
[/dl]

### 色彩視覺的三個恆常現象

[flow]
1. 色彩恆常性 — 同一物體在不同光線下，大腦會自動補正為「正確」顏色（白紙在黃光下仍被感知為白色）
2. 同時對比效應 — 同一灰色在深色背景上顯得淺，在淺色背景上顯得深
3. 色彩疲勞效應 — 長時間看某種顏色後，感知會偏移到補色方向
[/flow]

> **「調色疲勞」是調色師最危險的職業病**
> 長時間盯著同一畫面調色，眼睛會對當前色調適應，導致過度校正。
>
> 專業調色師的習慣：每 30-45 分鐘休息 5 分鐘，或用「中性灰眼神飛秒測試」——閉眼 30 秒後快速看畫面，感受第一眼的色偏印象。

[callout warning title="避免調色疲勞的實用技巧]
- 每隔 30 分鐘起身走動，讓眼睛遠離螢幕
- 準備一張標準色卡紙放在旁邊，偶爾看一眼作為「色彩重置」
- 不要在疲勞時做最終確認——隔天早上再看往往會發現問題
- 使用 DaVinci 的「A/B 比較」功能，切換回原始畫面快速重置視覺
[/callout]

## 數位影像的色彩模型

### RGB 色彩空間

```prompt [label="RGB 色彩模型的實際運作"]
每個像素有 R（紅）、G（綠）、B（藍）三個通道值

8-bit：  每通道 256 階（0-255），共 16.7 百萬色
10-bit：每通道 1024 階，共 10.7 億色
12-bit：每通道 4096 階，共 680 億色
16-bit：每通道 65536 階，用於 HDR 後製

調色時的意義：
  R=255, G=255, B=255 → 純白
  R=0,   G=0,   B=0   → 純黑
  R=255, G=0,   B=0   → 純紅
  R=255, G=165, B=0   → 橙色
```

### 色域（Color Gamut）與色彩空間

色域決定了一個系統能表現的色彩範圍：

[tags]
- [blue] sRGB / Rec.709：標準螢幕色域，涵蓋可見光的約 35%，適合網頁與一般影片
- [orange] DCI-P3：電影院標準，約為 sRGB 的 1.5 倍，Apple 顯示器與高階手機使用
- [green] Rec.2020：HDR 標準，涵蓋可見光的約 75%，廣播電視 HDR 標準
- [purple] ACES：學院色彩編碼系統，完整涵蓋所有攝影機可拍攝的色彩範圍
[/tags]

### Gamma 曲線與感知亮度

人眼對亮度的感知是非線性的——對暗部變化更敏感，對亮部變化相對不敏感：

```prompt [label="Gamma 曲線的本質"]
線性光（Linear Light）：攝影機感光元件的真實反應，光量翻倍 = 數值翻倍
人眼感知（Perceptual）：非線性，暗部辨識力更高，需要更多 bit 保存暗部細節

Gamma 2.2（sRGB）：電腦螢幕標準
Gamma 2.4（Rec.709）：電視廣播標準
LOG Gamma：高反差壓縮，保留最大動態範圍用於後製
```

## LOG 格式：專業攝影機的秘密

### 為什麼要用 LOG 拍攝

LOG（對數）格式是攝影機廠商設計的一種特殊 Gamma 曲線，目的是在影像感光器有限的 bit depth 內，盡可能保留最大的動態範圍：

[flow]
1. 原始場景可能有 14-16 stops 的動態範圍（高光到陰影的亮度比）
2. 標準 Rec.709 只能記錄約 6-8 stops
3. LOG 用對數壓縮，把 14 stops 的資訊塞進 10-12 bit 的影像檔案
4. 代價是：直接看起來像「洗白」「低對比」「灰矇矇」
5. 調色時還原 Gamma，再做風格化處理
[/flow]

### LOG 與 Rec.709 視覺對比

[compare label-left="LOG 原始畫面" label-right="套用 Rec.709 轉換後"]
- 低對比，畫面灰矇矇 | 對比正常，暗部紮實高光通透
- 色彩平淡，飽和度極低 | 色彩飽滿鮮明
- 保留 14+ stops 動態範圍 | 約 6-8 stops 可用範圍
- 不適合直接觀看或交付 | 適合直接輸出到 YouTube / 電視
- 必須經過色彩轉換才能調色 | 可直接進行風格化調色
[/compare]

### 各廠商 LOG 格式對照

[tags]
- [blue] Sony S-Log2 / S-Log3：Sony Alpha 與攝影機系列，S-Log3 動態範圍更大
- [orange] Panasonic V-Log / V-Log L：GH5/GH6/S5II 系列，V-Log L 為 V-Log 的輕量版
- [green] Canon Log 2 / Log 3：Cinema EOS 系列，C70/C300 Mark III
- [purple] DJI D-Log / D-Log M：大疆無人機與運動相機，M 版保留更多亮部
[/tags]

```prompt [label="在 DaVinci 正確處理 LOG 素材"]
方法一：Color Space Transform（CST）節點
→ 節點庫搜尋 CST → 設定輸入色彩空間（如 S-Gamut3.Cine / S-Log3）
→ 設定輸出色彩空間（Rec.709 / Gamma 2.4）
→ 此節點放在節點樹最前端

方法二：在 Color 管理中設定（推薦新手）
→ 專案設定 → 色彩管理 → 輸入色彩空間設為攝影機 LOG
→ 時間軸色彩空間設為 Rec.709
→ DaVinci 自動轉換所有素材

方法三：套用官方提供的技術 LUT
→ 各廠商提供 LOG → Rec.709 的 LUT 檔案（可官網免費下載）
→ 在調色節點最前端套用此 LUT
```

[callout tip title="新手建議]
第一次接觸 LOG 素材時，建議先用「方法二」在 Color 管理中設定自動轉換。這樣你不需要在每個鏡頭手動處理技術轉換，可以專注在創意調色上。等熟悉後再改用 CST 節點獲得更精確的控制。
[/callout]

# 工作環境：Color 頁面與示波器
> 示波器是調色師的眼睛——不懂示波器，就像外科醫生沒有 X 光

## Color 頁面完整介面解析

### 六個工作區域

[flow]
1. 節點編輯器（Node Editor）— 畫面下方，調色邏輯的核心
2. 調色工具列（Color Panels）— 左側面板：Color Wheels、Curves 等主要工具
3. 示波器面板（Scopes）— 右側，波形圖、向量圖等診斷工具
4. 素材縮略圖（Gallery）— 中間上方，存放調色靜態畫面（Stills）
5. 監視器（Viewer）— 中間，即時預覽調色結果
6. 畫廊（Gallery）— 管理調色預設與 Stills，跨鏡頭比較用
[/flow]

### Waveform（波形圖）

波形圖是最基礎也最重要的示波器。橫軸對應畫面的左右位置，縱軸是亮度值（0-100%）：

```prompt [label="波形圖解讀指南"]
讀取方式：把波形圖想成把畫面「側過來看」

0%（底部）   = 畫面中最暗的區域（理論純黑）
100%（頂部） = 畫面中最亮的區域（理論純白）

廣播安全範圍：7.5% - 100%（Rec.709 廣播標準）
一般螢幕範圍：0% - 100%

常見問題：
  波形集中在底部 → 整體曝光不足，需要提亮
  波形超過 100%   → 過曝，高光細節丟失（無法救回）
  波形只在 20-70% → 對比度不足，畫面「平」
```

> **「死白」與「死黑」**
> 當波形觸及 0% 下方（死黑）或 100% 以上（死白），代表該區域像素完全沒有細節。
>
> 死白是拍攝時的問題（曝光過度），調色無法救回。死黑則在調色時可以適當利用——電影感調色往往刻意保留一點「純黑」區域增加深度感。

[callout warning title="不可逆的資料損失]
一旦拍攝時高光過曝（死白），該區域的像素資訊已經完全丟失，調色軟體無能為力。這就像在錄音時爆音一樣，後製無法恢復原本的聲音。所以拍攝時寧可欠曝一點（往左偏），也不要過曝——欠曝的暗部至少還保有可以拉回的資訊。
[/callout]

### Parade（分量圖）

Parade 圖同時顯示 RGB 三個通道的獨立波形，是判斷色偏的最直接工具：

[flow]
1. 三條線等高 → 中性色，無色偏（白色物體應呈現此狀態）
2. R 通道偏高 → 整體偏紅（暖色偏）
3. B 通道偏高 → 整體偏藍（冷色偏）
4. R 高 G 低 B 低 → 偏紅
5. R 低 G 低 B 高 → 偏藍
6. R 低 G 高 B 低 → 偏綠
[/flow]

```prompt [label="用 Parade 圖判斷色偏範例"]
場景：在鎢絲燈光下拍攝的訪談

看到 Parade 圖：R 通道比 G、B 高 10-15%
→ 整體偏暖橘色（鎢絲燈白平衡未校正）
→ 解法：在 Offset（全域）中往青藍方向微調
→ 調整到 RGB 三條線等高，白色物體變中性白
```

### Vectorscope（向量圖）

向量圖顯示色相（Hue）和飽和度（Saturation）的分布，呈圓形：

[tags]
- [blue] 圓心位置 = 中性灰/白/黑（無色彩）
- [orange] 距圓心距離 = 飽和度（越遠越飽和）
- [green] 角度方向 = 色相（圓圈上的 R/G/B/Cy/Mg/Yl 六個目標點）
- [purple] 膚色線（Skin Tone Line）= 從圓心到右側約 10-11 點鐘方向的參考線，健康膚色應落在此線附近
[/tags]

### Histogram（直方圖）與 CIE 色度圖

```prompt [label="Histogram 讀取要點"]
橫軸：亮度（0=黑，255 或 1023=白）
縱軸：該亮度的像素數量

理想人像畫面：
  鐘形分布，中間調最多像素
  暗部（左側）略有分布代表陰影細節存在
  高光（右側）輕微分布代表適當高光

過曝跡象：直方圖右側被截斷（有很高的牆）
欠曝跡象：直方圖左側被截斷

CIE 色度圖（Chromaticity Diagram）：
  顯示色彩的絕對色度位置，用於判斷是否超出目標色域
  在 HDR 與廣播工作中重要
```

# 節點系統：DaVinci 調色的核心架構
> 節點是 DaVinci 最強的設計，也是大多數人最先放棄的地方——理解它，一切迎刃而解

## 節點的基本概念

### 圖層思維 vs 節點思維

[compare label-left="Photoshop 圖層思維" label-right="DaVinci 節點思維"]
- 上方圖層蓋住下方圖層 | 像水管，資訊從左流向右
- 疊加方向從下到上 | 每個節點做一件事
- 適合靜態影像處理 | 可做分支、混合、非線性處理
- 效果疊加難以追蹤 | 節點樹一目了然
[/compare]

[callout tip title="新手轉換心法]
剛從 Photoshop 或手機修圖 App 轉入 DaVinci 的調色師，最容易卡在「圖層思維」。記住：節點不是圖層，沒有「誰蓋誰」的問題。每個節點只負責「修改進入的影像，然後送出去」。把節點想成一條流水線，而不是一疊紙。
[/callout]

```prompt [label="節點的基本結構"]
每個節點有：
  輸入端（左側三角）：接收上個節點的影像資訊
  輸出端（右側三角）：傳遞處理後的影像到下個節點
  Key 輸入（左下角小三角）：接收 Alpha 遮罩資訊
  Key 輸出（右下角小三角）：輸出 Alpha 遮罩資訊

顏色編碼：
  藍色邊框 = 目前選取的節點
  橘色標籤 = 節點有調色資訊（有做調整）
  灰色標籤 = 節點為空（尚未調整）
```

## 四種核心節點類型

[tabs]
[tab label="串行 Serial"]
- 影像依序通過每個節點，最基本也最常用
- 適合：LOG 轉換、一級校正、對比調整、風格 LUT
- 特點：簡單直覺，一個節點做一件事
[/tab]
[tab label="並行 Parallel"]
- 多條路徑同時處理同一個輸入，結果自動混合
- 適合：分開處理高光與陰影、創意風格比較
- 特點：各調整互不影響，可獨立控制強度
[/tab]
[tab label="圖層 Layer"]
- 上層蓋下層，支援混合模式（Overlay、Soft Light 等）
- 適合：需要 Photoshop 風格疊加效果的場景
- 特點：可以做複雜的多重曝光或質感疊加
[/tab]
[tab label="外部 Outside"]
- 遮罩選到的走一條路，沒選到的走另一條路
- 適合：天空偏藍但不影響地面、主體與背景分開處理
- 特點：自動分離遮罩內外，不需要手動複製節點
[/tab]
[/tabs]

### 串行節點（Serial Node）

最常用的基本節點，影像依序通過每個節點：

[flow]
1. 節點 1：LOG 轉換（CST 或技術 LUT）
2. 節點 2：一級校正（曝光、白平衡）
3. 節點 3：對比度與 S 形曲線
4. 節點 4：二級調整（膚色、特定顏色）
5. 節點 5：創意風格（LUT 或整體色調）
6. 節點 6：輸出限制（防止過曝或欠曝）
[/flow]

> **「一個節點做一件事」原則**
> 把一級校正、對比調整、膚色處理分開到不同節點，不要全部堆在一個節點裡。
>
> 好處：可以單獨關閉某個節點做 A/B 比較；可以單獨複製某個節點到其他鏡頭；節點樹一目了然，六個月後重新打開仍能看懂自己在做什麼。

### 並行節點（Parallel Node）

並行節點同時處理同一個輸入，結果自動混合（平均）：

```prompt [label="並行節點應用場景"]
應用一：分開處理高光與陰影
  並行路徑 A：用 Power Window 遮罩只調暗部
  並行路徑 B：用 Power Window 遮罩只調高光
  → 自動混合，兩種調整互不影響

應用二：創意風格比較
  並行路徑 A：橙青風格
  並行路徑 B：電影感冷調
  → 分別套用 LUT，用混合強度（Blend）比例決定混合量

操作方式：
  右鍵節點 → 「在後方加入並行混合」
  → 出現並行節點輸入器（Parallel Mixer）
  → 將新節點連接到並行輸入器
```

### 圖層節點（Layer Node）

圖層節點和並行節點類似，但合成方式是「上層蓋下層」而非平均混合，支援混合模式（Blend Mode）：

[tags]
- [blue] 正常（Normal）：上層以透明度覆蓋下層，最常用
- [orange] 疊加（Overlay）：增加對比與色彩飽和度，類似 Photoshop 疊加
- [green] 柔光（Soft Light）：柔和版本的疊加，效果更細膩
- [purple] 乘法（Multiply）：讓兩個影像相乘，只有亮部才有效果
[/tags]

### 外部節點（Outside Node）

外部節點的作用是：讓遮罩（Qualifier/Power Window）選到的區域走一條路，沒選到的區域走另一條路：

```prompt [label="外部節點實際運用"]
場景：要讓天空偏藍，但不影響地面

節點 A：Qualifier 選取天空（藍色範圍）
         ↓ 調整天空：飽和度 +30、偏藍
         
節點 B（外部節點）：自動接收「天空以外」的部分
                    ↓ 對地面做補充調整（暖色 +10）

操作：
  右鍵節點 A → 「加入外部節點」
  → 自動建立外部節點，Key 輸出連接到外部節點的 Key 輸入
  → 外部節點只影響未被遮罩選取的區域
```

# 一級調色：讓畫面技術上正確
> 調色分兩個階段：先讓畫面「正確」，再讓它「好看」——這章解決「正確」的問題

## 系統化一級調色流程

### 五步驟技術校正

[flow]
1. LOG 轉換 → 確認節點 01 正確套用了 CST 或技術 LUT，素材看起來不再灰矇矇
2. 曝光評估 → 看波形圖，確認主體亮度在 50-70%，高光不超過 100%
3. 色偏校正 → 看 Parade 圖，調整讓中性色（白牆、灰卡）的 RGB 三線等高
4. 對比建立 → 拉 S 形曲線，讓暗部紮實、高光有層次
5. 最終確認 → 再次看波形、Parade、Vectorscope 確認所有數值在安全範圍
[/flow]

[reveal title="新手最常見的順序錯誤"]
很多新手會先調整對比或風格，結果發現怎麼調都不對。原因是：**在素材還未達到技術標準前做任何風格化調整，都是在沙地上蓋房子**。

正確的順序永遠是：技術校正（曝光、白平衡、對比）→ 風格創作（LUT、色彩偏移）。前者讓畫面「正確」，後者讓畫面「好看」。兩者順序顛倒會事倍功半。
[/reveal]

## Color Wheels 精確操作

### Lift / Gamma / Gain / Offset 的定義

[tags]
- [blue] Lift（暗部）：影響 0-50% 亮度範圍，改變最暗區域的色彩與亮度
- [orange] Gamma（中間調）：影響 30-70% 亮度範圍，是調整觀感最有效的控制點
- [green] Gain（高光）：影響 50-100% 亮度範圍，控制最亮區域的色彩與亮度
- [purple] Offset（全域偏移）：均等地影響所有亮度範圍，最適合快速修正整體色偏
[/tags]

```prompt [label="Color Wheels 精確操作技巧"]
控制方式：
  拖曳色輪中心點 → 改變色相偏向（往哪個顏色偏移）
  旋轉色輪下方圓圈 → 改變亮度值（往上提亮 / 往下壓暗）

精細調整技巧：
  按住 Ctrl/Cmd 拖曳 → 緩慢微調（1/4 速度）
  點擊數值欄位 → 直接輸入精確數值
  右鍵數值 → 重置到 0

Log Wheels（對數控制輪）：
  和 Color Wheels 類似，但每個控制點影響的範圍更精確
  Shadow（陰影）/ Midtone（中間調）/ Highlight（高光）
  比 Lift/Gamma/Gain 更適合精細處理 LOG 素材
```

### 白平衡精確校正技巧

```prompt [label="三種白平衡校正方法"]
方法一：自動白平衡（快速但不精確）
  Color Wheels 右下角 → 點擊白平衡吸管圖示
  → 在監視器點擊應為中性白的區域
  → DaVinci 自動計算 Temp / Tint 補正

方法二：Parade 圖手動校正（最精確）
  目標：找一個應為中性色的物體（白牆、灰卡、白色 T 恤）
  → 看 Parade 圖中該物體對應的區域，三條線是否等高
  → 在 Offset 中往等高方向調整
  → 反覆確認直到中性物體的 RGB 值相等

方法三：Color Temperature / Tint 控制（直觀）
  一般工具 → Temperature（色溫）/ Tint（色調）
  Temperature 往左 = 偏藍冷 / 往右 = 偏橙暖
  Tint 往左 = 偏綠 / 往右 = 偏洋紅
  適合知道目標色溫（日光 5500K、陰天 6500K、鎢絲燈 3200K）時快速設定
```

[compare label-left="快速方法" label-right="精確方法"]
- 自動白平衡吸管一鍵校正 | Parade 圖手動校正 RGB 三線
- 適合時間緊迫、畫面簡單的場景 | 適合專業輸出、有中性色參考物的場景
- 可能因為畫面複雜而計算錯誤 | 結果穩定可重現，數據說話
- Temperature/Tint 滑桿直觀調整 | 用 Offset 微調，精度更高
[/compare]

[callout tip title="白平衡校正小技巧"]
如果畫面中沒有任何中性色物體可以参考，可以試試「灰卡技巧」：拍攝時在場景中放一張白紙或灰卡，調色時先以此為基準做白平衡校正，完成後再用 Qualifier 排除灰卡區域。這樣即使環境色複雜，也能獲得準確的白平衡起點。
[/callout]

## 對比度控制技術

### 什麼是「好的對比度」

對比度不是越高越好，而是要符合內容的情緒需求：

[flow]
1. 高對比度（Contrast > 1.0，Pivot 0.5）— 戲劇性、衝突感、恐怖/驚悚類型
2. 中對比度（Contrast 0.9-1.1）— 一般紀錄片、訪談、真實感
3. 低對比度（Contrast < 0.9，Lift 提亮）— 夢幻、懷舊、日系風格
4. 膠片對比度（S 形曲線，Lifted Blacks）— 電影感，暗部不完全死黑
[/flow]

```prompt [label="精確對比度控制的四種方法"]
方法一：Contrast + Pivot 滑桿
  Contrast：整體對比倍率
  Pivot：對比變化的中心點（設為 0.5 = 以中間灰為基準）

方法二：Lift + Gain 分別控制
  降低 Lift = 壓深陰影
  提高 Gain = 拉亮高光
  → 同時做 = 增加對比，適合精細控制兩端

方法三：Custom Curves 的 S 形曲線（最精確）
  見下一章「曲線工具」詳細說明

方法四：Shadow / Highlight 恢復
  高光恢復：Log Wheels 的 Highlight → 往下拉
  暗部恢復：Log Wheels 的 Shadow → 往上推
```

### 亮度範圍的廣播安全

```prompt [label="廣播安全亮度範圍設定"]
一般螢幕輸出（YouTube / Instagram）：
  亮度：0% - 100%
  允許超黑（Super Black）：否
  允許超白（Super White）：否

廣播電視（台灣廣電標準）：
  亮度：7.5% - 100%（NTSC 標準）或 0% - 100%（PAL）
  最高色彩訊號（Composite）：不超過 110 IRE

設定 DaVinci 安全保護：
  Display → Safe Color Warning（顯示超出安全範圍的紅色提示）
  在輸出節點使用 Curves 將最暗限制在 2%，最亮限制在 98%
```

# 曲線工具：精細控制每一個色彩維度
> Curves 是調色師的手術刀——Color Wheels 是斧頭，Curves 是解剖刀

## Custom Curves（自訂曲線）

### 曲線的基本邏輯

曲線圖橫軸是輸入值（原始畫面），縱軸是輸出值（調色後畫面）。對角線是「未調整」狀態：

[flow]
1. 把曲線往左上拖 → 提亮（同樣的輸入，輸出更亮）
2. 把曲線往右下拖 → 壓暗（同樣的輸入，輸出更暗）
3. 在暗部往上 + 高光往下 → S 形曲線，增加對比
4. 在暗部往上（提亮暗部）→「Lifted Blacks」電影感
5. 在曲線底部打一個固定點，在高光拉高 → 只提亮高光，不動暗部
[/flow]

### S 形曲線的電影感應用

```prompt [label="電影感 S 形曲線具體操作"]
步驟 1：在暗部（約 25% 位置）加一個控制點，往下拉 5-8%
         → 讓陰影更有深度感

步驟 2：在高光（約 75% 位置）加一個控制點，往上拉 3-5%
         → 讓高光更通透

步驟 3：在最暗處（0% 附近）加一個固定點，往上提 2-5%
         → Lifted Blacks：讓陰影不完全死黑，產生膠片感

步驟 4：在最亮處（100% 附近）加一個限制點，輕微往下
         → 壓制高光，防止過曝同時增加電影感的高光壓制

注意事項：
  控制點不超過 4-5 個，太多控制點曲線會震盪
  使用「平滑（Smooth）」選項讓曲線過渡更自然
```

[callout warning title="S 形曲線常見陷阱]
過度拉 S 形曲線是新手最常犯的錯誤——暗部拉太深導致死黑，高光拉太高導致死白。每次調整後務必看波形圖確認：暗部不低於 2%，高光不超過 98%。好的 S 形曲線是讓畫面「有對比」而不是「有死黑死白」。
[/callout]

### 分通道曲線（YRGB Curves）

DaVinci 的曲線允許獨立控制 Y（亮度）、R、G、B 四個通道：

[tags]
- [blue] Y 通道（亮度曲線）：只調整亮度，不影響色相與飽和度，最乾淨的亮度控制
- [orange] R 通道：往上提亮紅色 / 往下壓暗青色（紅的補色）
- [green] G 通道：往上提亮綠色 / 往下壓暗洋紅
- [purple] B 通道：往上提亮藍色 / 往下壓暗橙色（金黃色）
[/tags]

```prompt [label="分通道曲線的創意應用"]
創造暖色高光 + 冷色陰影（電影常見手法）：
  R 通道：在高光位置（75%）微微往上，在暗部（25%）微微往下
  B 通道：在暗部（25%）微微往上，在高光（75%）微微往下
  → 結果：高光偏暖橙，陰影偏冷藍（即 Orange & Teal 的基礎）

創造綠色陰影（恐怖、科技感）：
  G 通道：在暗部（15%）往上拉
  → 讓陰影帶有綠色調，搭配冷色整體色調

創造夕陽感高光：
  R 通道：高光（80%+）大幅往上
  B 通道：高光（80%+）往下壓
  → 高光呈現橙紅色，像夕陽光照效果
```

## Hue vs. 系列曲線（六種 HSL 曲線）

這是 DaVinci 最強大的二級調色工具之一，讓你可以針對特定色相做精確調整：

### Hue vs. Hue（色相轉移）

輸入色相→輸出色相的轉換，可以把某個顏色「推向」另一個方向：

```prompt [label="Hue vs. Hue 實際應用"]
場景：秋天拍攝，希望樹葉更偏金黃（原本偏綠黃）

步驟：
1. 在 Hue vs. Hue 曲線找到黃綠色的位置
2. 用吸管在監視器點取要調整的顏色（自動定位到該色相）
3. 把控制點往橙黃方向拖（往右 = 往橙紅方向偏移）

常見應用：
  天空偏青 → 往藍色方向推（更乾淨的天空藍）
  草地偏黃 → 往綠色方向推（更鮮活的草綠）
  膚色偏橙 → 微微往黃方向拉（更自然的東方膚色）
```

### Hue vs. Saturation（選色飽和度）

針對特定色相的飽和度控制，不影響其他顏色：

```prompt [label="Hue vs. Saturation 常見用法"]
降低整體飽和度但保留膚色：
  1. 整體：Saturation 滑桿降至 70-80%（整體去飽和）
  2. Hue vs. Sat：找到膚色（橙黃）位置，把控制點往上拉
  → 結果：背景色彩淡雅，但主體膚色仍然飽滿

強調天空的藍色：
  找到天空藍色位置 → 往上拉
  → 天空變得更飽和，其他顏色不受影響

消除背景的雜色：
  找到問題顏色（如偏洋紅的牆壁）→ 往下拉至 0
  → 背景顏色更乾淨，視覺焦點更突出
```

### Hue vs. Luminance（選色亮度）

針對特定色相的亮度控制，可以讓特定顏色的物體變亮或變暗：

[tags]
- [blue] 提亮天空中的白雲（白色在亮度上分布）
- [orange] 壓暗過亮的黃色路燈，讓夜景更有層次
- [green] 讓草地的綠色稍微壓暗，與天空亮度拉開差距
- [purple] 用亮度區分兩個顏色相近但亮度不同的物體
[/tags]

### Luminance vs. Saturation（亮度與飽和度）

控制不同亮度區域的飽和度——例如：讓高光的顏色淡化，讓陰影不要過度飽和：

```prompt [label="Luminance vs. Saturation 電影感設定"]
電影感常用設定：
  高光端（右側）：把飽和度往下拉 15-25%
  → 模擬膠片高光失飽和的特性，讓高光看起來更「柔和」

  暗部端（左側）：把飽和度往下拉 10-20%
  → 讓陰影不要有過分強烈的顏色，增加深沉感
```

# 二級調色：限定器、Power Window 與追蹤
> 一級調色是整體，二級調色是局部——精準控制每一個你想改變的區域

## Qualifier（限定器）

### 限定器的三種模式

[flow]
1. HSL（色相/飽和度/亮度）限定 — 根據顏色特徵選取，最常用
2. RGB 限定 — 根據 R/G/B 通道數值選取，適合邊界清晰的顏色
3. 3D 限定 — 在三維色彩空間精確選取，適合複雜場景
[/flow]

### HSL 限定器精確操作

```prompt [label="HSL 限定器完整操作流程"]
步驟 1：啟用限定器
  在 Color 頁面點擊「限定器（Qualifier）」圖示
  工具列切換到 HSL 模式

步驟 2：選取目標顏色
  用吸管（Eyedropper）在監視器點擊目標顏色
  按住 Shift 點擊 = 增加選取範圍
  按住 Alt 點擊 = 縮小選取範圍（移除某顏色）

步驟 3：在 Highlight 模式下查看遮罩品質
  按 Shift + H → 遮罩顯示：被選取區域=白色，未選取=黑色
  調整 H（色相）/ S（飽和度）/ L（亮度）範圍滑桿優化遮罩

步驟 4：使用 Matte 精修工具
  Shrink/Grow（縮小/擴展）：讓遮罩邊緣內縮或外擴
  Blur（柔化）：讓遮罩邊緣更柔和，避免硬邊
  Denoise（降噪）：消除遮罩中的噪點
  Clean Black / Clean White：分別清理遮罩的黑色或白色區域

步驟 5：開始二級調整
  確認 Highlight 模式關閉（Shift + H 切換）
  現在所有 Color Wheels / Curves 操作都只影響被遮罩選取的區域
```

### 限定器常見問題排除

```prompt [label="限定器問題排解方案"]
問題：邊緣選取不乾淨，漏選了不該選的部分
解法：
  1. 用 Shrink 縮小遮罩範圍
  2. 增加 Denoise 降噪強度
  3. 考慮用 Power Window 限制生效範圍（與限定器組合使用）

問題：膚色選取時，同色系的木頭家具也被選到了
解法：
  1. 在 HSL 滑桿中收窄 H（色相）範圍，讓選取更精準
  2. 用 Power Window 只在人臉區域生效
  3. 使用 3D 限定器，能在色彩空間三維方向更精確選取

問題：人物移動後，之前選取的區域不再準確
解法：
  → 需要使用動態追蹤（見下方「追蹤」章節）
```

## Power Window（動態遮罩）

### 五種 Power Window 形狀

[tags]
- [blue] 圓形（Circle）：人臉亮度提升、圓形物體強調
- [orange] 矩形（Rectangle）：天空、地板等大範圍水平區域
- [green] 多邊形（Polygon）：不規則形狀的精確遮罩繪製
- [purple] 曲線（Curve）：使用貝茲曲線繪製最複雜的形狀
[/tags]

```prompt [label="Power Window 精確操作技巧"]
基本操作：
  點擊 Power Window 圖示 → 選擇形狀 → 在畫面拖拉建立遮罩
  
  外圓（硬邊）= 遮罩生效的實際邊界
  內圓（軟邊）= 遮罩從 100% 效果漸變到 0% 的過渡區域
  → 拉大軟邊範圍 = 更自然的邊緣過渡

進階技巧：
  多個 Power Window 組合：
    同一個節點可以加多個 Power Window
    預設是「相加」關係
    按住 Ctrl 點擊第二個遮罩 = 「相減」（從第一個遮罩挖掉）

  反轉遮罩：
    點擊 Power Window 面板右上角的「反轉」按鈕
    → 讓生效區域與原本相反（選形狀外面）

  羽化（Softness）細節控制：
    H/V Soft：水平/垂直方向的羽化程度
    O/I Soft：外部/內部的羽化程度
    → 精細控制遮罩各個方向的柔化程度
```

### Power Window 進階應用

```prompt [label="Power Window 電影感光影塑造"]
技巧一：中心亮邊緣暗（暈影 Vignette）
  建立圓形 Power Window → 設定大軟邊 → 反轉
  → 外部節點降低亮度/飽和度
  → 視線自然聚焦畫面中心

技巧二：獨立提亮主體
  用多邊形 Power Window 繪製主體輪廓（人臉、商品）
  → 在遮罩內提亮 Gamma +0.1 到 +0.2
  → 配合柔化邊緣，讓主體在場景中自然突出

技巧三：天空漸層（Natural Gradient）
  建立矩形 Power Window → 只覆蓋上半部天空
  → 設定大軟邊在天空與地平線交界處
  → 單獨調整天空飽和度、藍色成分、壓制高光
```

# 膚色處理：人物調色的關鍵技術
> 觀眾對膚色最敏感——膚色不對，整個畫面的真實感就崩塌了

## 膚色的生理基礎

### 膚色為什麼難處理

人類皮膚的色彩構成非常複雜：

[flow]
1. 黑色素（Melanin）— 決定膚色深淺，從淺粉到深褐
2. 血紅素（Hemoglobin）— 血管分布帶來的紅色，在臉頰、嘴唇、耳朵特別明顯
3. 光線散射（Subsurface Scattering）— 光線進入皮膚後散射，讓皮膚看起來「通透」而非不透明
4. 環境反射 — 周圍環境顏色反射在皮膚表面，是最常見的膚色問題來源
[/flow]

### 向量圖與膚色線

```prompt [label="用向量圖判斷膚色品質"]
膚色線（Skin Tone Line）：
  從向量圖圓心，延伸到 11 點鐘方向（約 10-12 點方向）的一條參考線
  所有健康的人類膚色（無論深淺）都應該落在這條線附近

膚色正常：向量圖中膚色的點群落在膚色線附近
偏紅（發炎感）：點群偏向膚色線右側（往紅色方向）
偏黃（蠟黃感）：點群偏向膚色線左側（往黃色方向）
偏洋紅：點群往上偏移
偏橙：點群往右下偏移

判斷步驟：
1. 在監視器框選人臉區域（設定 ROI）
2. 看向量圖中點群的分布位置
3. 判斷需要往哪個方向調整
```

## 膚色校正技術

### 用 Qualifier 精確選取膚色

```prompt [label="膚色選取的最佳實踐"]
挑戰：膚色選取容易「誤傷」
  → 皮膚顏色與木頭、沙子、部分磚牆相近
  → 選取膚色時常常連背景也一起選到

解決方案：組合使用限定器 + Power Window
  步驟 1：建立限定器節點，HSL 選取膚色範圍
  步驟 2：在同一節點建立多個 Power Window 覆蓋臉部、頸部、手部
  步驟 3：限定器和 Power Window 同時作用 = 只在臉部/手部範圍內的膚色才被選取
  步驟 4：啟用追蹤，讓 Power Window 跟著人物移動

HSL 限定器膚色範圍建議：
  H（色相）：橙色範圍，約 15°-45°
  S（飽和度）：避免選到白牆（低飽和），設在 15-80%
  L（亮度）：避免選到陰影死黑，設在 15-85%
```

### 不同膚色的調整策略

```prompt [label="膚色調整的方向性建議"]
淺膚色（歐美白人）：
  常見問題：在特定光線下顯得過粉或過紅
  調整方向：Hue vs. Hue 把偏紅的膚色往橙黃方向微推 5-10°
  飽和度：Hue vs. Sat 降低膚色飽和度 10-15%，防止「紅臉」感

中等膚色（東亞、拉丁裔）：
  常見問題：黃色成分過重，顯得「蠟黃」
  調整方向：Hue vs. Hue 把偏黃的膚色往橙色方向微推
  亮度補正：Luminance vs. Luminance 在膚色亮度範圍微微提亮
  
深膚色（非裔、南亞）：
  常見問題：在錯誤曝光下喪失細節，或在補光不足時完全沒有紋理
  調整方向：謹慎提亮陰影，保留皮膚紋理；飽和度通常不需降低
  重要：Log Wheels 的 Shadow → 微微提亮，保留暗部細節

所有膚色的通用規則：
  Vectorscope 向量圖確認膚色點群落在膚色線上
  波形圖確認臉部亮度在 50-75%（特寫訪談的標準亮度）
  不要為了「讓膚色好看」而破壞整體場景的色彩一致性
```

[callout tip title="膚色處理心法]
處理膚色時，先用 Vectorscope 確認方向（偏紅/偏黃），再用 Qualifier 只選膚色範圍微調。記住：觀眾對膚色的敏感度遠超過其他色彩——膚色正確了，整個畫面的真實感就會大幅提升。如果時間有限，膚色校正的投資報酬率最高。
[/callout]

### 美化技術與自然感的平衡

[tags]
- [blue] 輕微提亮臉部（Gamma +0.08 到 +0.12）：讓主角從背景中脫穎而出
- [orange] 降低膚色飽和度（Hue vs. Sat -10 到 -15%）：去除「紅臉」的不健康感
- [green] 暗部提亮（Lift +0.03）：減少眼窩、法令紋的強烈陰影
- [purple] Color Warper 膚色微調：在色彩空間中把膚色精確推向最理想的方向
[/tags]

> **「美化」的邊界在哪裡？**
> 調色的目的是讓人看起來「最好的自己」，而不是「不像自己」。
>
> 一個好用的測試：把調色前後並排，請另一個人看，問他「這兩張照片是同一個人嗎？」如果答案是「不確定」，你的美化已經超過邊界。調色師負責皮膚的光澤與色彩，不負責移除真實的特徵。

# 視覺風格：從電影感到個人調色語言
> 技術讓畫面「正確」，風格讓畫面「難忘」——這章你開始建立自己的調色語言

## 電影感調色的核心要素

### 什麼讓調色有「電影感」

電影感不是一個調色參數，而是幾個技術選擇的組合：

[flow]
1. Lifted Blacks（暗部提亮）— 讓陰影從純黑提到深灰，模擬膠片的暗部特性
2. Compressed Highlights（高光壓制）— 讓高光從純白壓下來，防止高光「死白」
3. 降低飽和度 — 膠片時代的色彩飽和度比數位低，整體降至 70-85%
4. Warmth Bias（暖色傾向）— 輕微暖色整體，使畫面更「有機感」
5. 粒狀感（Grain）— 模擬膠片顆粒，讓畫面不那麼「數位感」
[/flow]

```prompt [label="電影感基礎設定參數"]
一、Lifted Blacks：
  Custom Curves → 在底部（0%）打一個點往上提 3-5%
  → 讓最暗的純黑變成深灰

二、Compressed Highlights：
  Custom Curves → 在頂部（100%）打一個點往下壓 2-4%
  → 讓高光略微壓制，防止死白

三、飽和度降低：
  Color Wheels → Saturation 設為 70-80
  → 整體色彩淡化，更接近膠片色感

四、暖色偏移：
  Gain → 微微往橙黃方向（約 1-2 個單位）
  → 讓高光帶有輕微暖意

五、膠片顆粒：
  ResolveFX 效果庫 → Film Grain
  → ISO 選擇 400-800（電影常見感光度）
  → Strength：0.3-0.5（不要太強，要看不出來）
```

## 經典調色風格

### Orange & Teal（橙青對比）

好萊塢動作片最常見的調色方案，利用膚色橙色與補色青色的對比：

```prompt [label="Orange & Teal 系統性操作"]
原理：人類膚色偏橙，青色（Teal）是橙色的補色
兩者同時存在時，對比最強，視覺衝擊最大

步驟 1：整體暖色基礎
  Offset 或 Gain → 微微往橙橘色方向（+2 到 +5 單位）

步驟 2：陰影往 Teal 推
  Lift → 往青色（Cyan + 一點 Green）方向推
  → 讓暗部呈現冷藍綠色（Teal）

步驟 3：膚色強調
  建立限定器節點 → 選取膚色
  Hue vs. Hue → 把膚色往橙色方向推 5-10°
  Hue vs. Sat → 提高膚色飽和度 10-15%

步驟 4：整體去飽和（保留橙青）
  整體 Saturation 降至 75-85
  Hue vs. Sat → 提高橙色和青色的飽和度

步驟 5：建立 LUT
  Color 頁面頂部選單 → 「產生 LUT」→ 存成 .cube 格式
  → 下次可以直接套用這個風格
```

### 日系清新風格

```prompt [label="日系清新調色操作"]
特徵：高亮、低對比、柔和、暖白感

步驟 1：整體提亮
  Custom Curves → 中間調（50%）往上拉約 15-20%
  Offset 整體提亮 +0.05 到 +0.10

步驟 2：Lifted Blacks（淺灰色陰影）
  Custom Curves → 底部固定點大幅上提（10-20%）
  → 讓陰影變淺灰而非黑色

步驟 3：去飽和 + 暖調
  Saturation 降至 60-75
  Gain 往暖黃色方向微推
  → 整體呈現暖白的「空氣感」

步驟 4：消除高對比邊緣
  高光壓制：Gain 往下微調，或 Curves 高光端下壓
  → 避免任何強烈的明暗對比，保持均勻柔和感

步驟 5：薄霧感（選用）
  效果庫 → Mist / Glow
  → Strength 0.1-0.2，Threshold 高一點（只讓高光泛光）
```

### 復古膠片風格（1970s Film Emulation）

```prompt [label="復古膠片調色操作"]
特徵：橙棕色調、褪色、粒狀、低飽和

步驟 1：整體橙褐色偏移
  Offset 整體往橙褐色方向
  Lift 往橙黃方向
  → 整個畫面偏向暖棕色

步驟 2：大幅 Lifted Blacks
  Curves 底部大幅提亮（15-25%）
  → 讓最暗的陰影也帶有橙褐色調，不存在純黑

步驟 3：壓低高光
  Gain 微微下壓，同時保持暖色
  → 高光不刺眼，整體反差低

步驟 4：去飽和
  Saturation 降至 50-65
  → 色彩「褪色」的老照片感

步驟 5：膠片顆粒（關鍵）
  Film Grain → ISO 800-1600
  Strength 0.5-0.7（比電影感更明顯）
  → 顆粒感是復古風格的靈魂

步驟 6：Hue vs. Hue 色相偏移
  藍色往青色方向推（讓天空偏舊照片的青藍）
  綠色往黃色方向推（草地偏黃褐色）
```

### 四種經典風格快速對照

[compare label-left="電影感" label-right="日系清新"]
- 對比度中高，暗部有深度 | 低對比，柔和均勻
- 飽和度 70-85% | 飽和度 60-75%
- 暖色高光 + 冷色陰影 | 整體暖白空氣感
- 適合動作片、劇情片 | 適合文青片、美食、旅拍
[/compare]

[compare label-left="Orange & Teal" label-right="復古膠片（1970s）"]
- 膚色橙 vs 背景青的強烈對比 | 整體橙棕色調，無純黑純白
- 飽和度 75-85%，保留橙青 | 大幅去飽和至 50-65%
- 好萊塢動作片標準 | 老照片、懷舊回憶
- LUT 強度 30-60% | 膠片顆粒 ISO 800-1600
[/compare]

## 製作個人調色 LUT


### LUT 的製作流程

```prompt [label="從調色到 LUT：完整製作流程"]
Step 1：建立一個「標準測試鏡頭」節點樹
  確保測試鏡頭包含：膚色、中性灰、高光、陰影、飽和色塊

Step 2：完成你的風格調色
  在測試鏡頭上完成所有想要的風格調整

Step 3：匯出 LUT
  Color 頁面選單 → 「Color 管理」→「產生 LUT」
  選擇 33 點 LUT（最常用）
  匯出格式：.cube（最通用的格式）

Step 4：在其他鏡頭測試 LUT
  新建節點 → 右鍵 → LUTs → 選取剛才的 .cube 檔
  測試不同類型素材（人像、風景、室內、夜景）
  確認 LUT 在各種場景下的效果穩定

Step 5：調整 LUT 強度
  在套用 LUT 的節點 → 右鍵 → Key → Opacity
  調整不透明度（50-80%）讓 LUT 效果更融合

注意事項：
  LUT 是在一級校正後套用，不是替代一級校正
  每次使用 LUT 後仍需做局部微調（膚色、問題高光等）
```

# 色彩管理：從攝影機到螢幕的完整鏈路
> 色彩管理是讓你在「所見即所得」環境下工作的關鍵基礎設施

## DaVinci 的色彩管理模式

### 三種色彩管理工作流程

[tags]
- [blue] 無管理（Unmanaged）：素材直接進入節點，你自己負責所有轉換，適合有經驗的調色師
- [orange] DaVinci YRGB Color Managed：DaVinci 自動處理輸入輸出色彩空間轉換，推薦一般工作流
- [green] ACES（Academy Color Encoding System）：電影院級別的科學色彩管理，適合高端製作
[/tags]

```prompt [label="DaVinci Color Managed 設定步驟"]
Step 1：專案設定 → 色彩管理
  色彩科學：DaVinci YRGB Color Managed

Step 2：設定輸入色彩空間
  Automatic（自動）：DaVinci 根據素材元數據自動偵測
  手動設定：若自動偵測錯誤，選取正確的攝影機格式

Step 3：設定時間軸色彩空間
  通常設為 DaVinci Wide Gamut / DaVinci Intermediate
  → 這是內部工作色彩空間，範圍最大

Step 4：設定輸出色彩空間
  Rec.709 Gamma 2.4（廣播電視、YouTube）
  DCI-P3（電影院投影）
  Rec.2020 ST2084 PQ（HDR 輸出）

效果：
  所有 LOG 素材自動轉換到可工作的色彩空間
  輸出時自動轉換到目標格式
  你只需要專注在創意調色，不需要手動做技術轉換
```

## ACES 工作流程入門

### ACES 的核心概念

ACES（Academy Color Encoding System）是美國電影藝術與科學學院制定的開放式色彩管理系統：

[flow]
1. 輸入：所有攝影機素材通過 IDT（Input Device Transform）轉換到 ACES 色彩空間
2. 工作：在 ACES 巨大的色彩空間內進行所有調色工作
3. 輸出：通過 ODT（Output Device Transform）轉換到目標顯示裝置
4. 優點：不同攝影機素材自動統一到同一個色彩空間，色彩一致性最好
[/flow]

# 職業工作法：效率、一致性與交付
> 調色技術只是工具，職業工作法決定你能不能可持續地做好的調色

## 場景間色彩匹配

### 分鏡色彩一致性

實際工作中，同一場景的不同鏡頭可能是在不同時間、不同光線條件下拍攝的，調色師需要讓它們看起來像在同一時間：

```prompt [label="場景色彩匹配工作流程"]
方法一：參考靜態畫面（Stills）比對
  Step 1：選取「基準鏡頭」（光線最好的那一顆）完成調色
  Step 2：右鍵 → 「截取靜態畫面（Grab Still）」
  Step 3：其他鏡頭調色時，把基準鏡頭的 Still 拉到比較視窗
  Step 4：根據 Still 調整直到兩個鏡頭的示波器數值接近

方法二：自動色彩匹配（Colour Match）
  同時選取需要匹配的鏡頭和目標鏡頭
  Color 頁面選單 → 「Colour Match」
  → DaVinci 自動計算需要的調整，作為初始校正基礎
  → 通常還需要手動微調，但節省大量時間

方法三：使用相同 PowerGrade（調色預設）
  把基準鏡頭的節點樹存成 PowerGrade（右鍵 Gallery → 新增 PowerGrade）
  其他鏡頭右鍵 → 套用 PowerGrade
  → 節點結構完全複製，再做針對性微調
```

## 批次調色與 PowerGrade

### 跨鏡頭效率工具

```prompt [label="批次調色操作方法"]
方法一：Group 節點（群組前/後節點）
  在 Clip 縮略圖右鍵 → 「加入到群組」→ 選取多個鏡頭加入同一群組
  Group Pre-Clip：在個別調色前加入的共用節點（如 LOG 轉換）
  Group Post-Clip：在個別調色後加入的共用節點（如輸出保護、整體風格）
  → 修改 Group 節點，所有群組內鏡頭同時更新

方法二：Remote Grades（遠端調色版本）
  一個鏡頭多次出現時（如 A/B Roll 同一素材用了多次）
  右鍵 Clip → Remote Grades → 啟用
  → 修改原始鏡頭的調色，所有使用該素材的地方同步更新

方法三：PowerGrade 庫管理
  把常用的節點樹存為 PowerGrade
  命名規範：[風格]-[適用素材]-[版本]
  例如：FilmLook-Sony-v2、SkinCorrect-Asian-A7IV
```

## 調色交付標準

### 輸出前的最終確認清單

```prompt [label="調色完成前的品質確認"]
技術面：
  □ 所有鏡頭的波形圖在廣播安全範圍（0-100%）
  □ Parade 圖確認中性物體（白牆、灰卡）白平衡正確
  □ 沒有任何鏡頭有未修正的嚴重色偏
  □ 場景間的色彩一致性（用 Stills 比對）
  □ 膚色在向量圖的膚色線附近

創意面：
  □ 整體調色風格符合導演/客戶的參考要求
  □ 情緒化場景的色彩強度符合敘事需要
  □ 夜景場景與日間場景的風格統一但有區別

輸出面：
  □ 確認輸出色彩空間（YouTube 用 Rec.709 / 電影用 DCI-P3）
  □ 確認是否需要輸出 LUT 檔案給客戶
  □ 交付 Still 庫（客戶要求修改時的參考基準）
```

### 與剪輯師/導演的協作

[flow]
1. 取得調色參考（Reference）— 客戶或導演提供的「目標影片」，作為風格方向
2. 建立調色意見交換流程 — 可用 DaVinci 的「Timeline Comment」或共享 Stills
3. 版本管理 — 使用 DaVinci 的「版本（Version）」功能保存多個調色版本
4. 遠端審片 — 輸出代理檔（1080p H.264）給客戶審閱，確認後再做最終輸出
[/flow]

# 總結

[summary]
- **色彩科學** | 理解 LOG 格式、色彩空間、人眼感知，建立調色的認知基礎，讓每個工具選擇有理論依據
- **示波器精通** | 波形圖、Parade、向量圖的系統解讀方法，用數據替代直覺，調色結果穩定可重現
- **節點系統** | 串行、並行、圖層、外部節點的完整掌握，設計可維護、可重複利用的專業節點樹架構
- **一級調色** | Color Wheels 精確操作、曝光校正、白平衡系統化流程，讓任何素材都達到廣播技術標準
- **曲線工具** | Custom Curves 與六種 Hue vs. 曲線的深度應用，對每一個色彩維度做精確的細膩控制
- **二級調色** | Qualifier 限定器、Power Window 遮罩、動態追蹤的完整技術鏈，實現局部精確調整
- **膚色處理** | 膚色的生理基礎、向量圖判斷、不同膚色的調整策略，讓人物在任何場景下都真實好看
- **視覺風格** | 電影感、橙青、日系、復古等經典風格的系統操作，並建立個人調色 LUT 形成一致風格語言
- **色彩管理** | DaVinci Color Managed 與 ACES 工作流程，確保色彩在從攝影機到螢幕的整個鏈路中正確
- **職業工作法** | 場景匹配、批次調色、PowerGrade 管理、交付標準，讓調色工作可持續、高效且專業
[/summary]
