## 各章節改善建議

### 章節一：入門認識
- 實作層為概念型章節，按 SKILL.md 規範可只有說明層，**無需修改**。
- 正面亮點：vote 互動暖身設計良好；compare 左右對照讓初學者立刻抓到差異；結尾 quiz 檢核概念清楚。

### 章節三：cc-switch
- 實作層薄弱：有 quiz 但缺乏「現在你試試切換」的動手引導。
- 建議：在「安裝與使用 cc-switch」卡片後加入一個 flow，引導學員動手操作：
  ```
  [flow]
  1. 確認 Claude Code 已安裝 — 終端機執行 `claude --version`
  2. 啟動 Claude Code — 執行 `claude`
  3. 在對話中切換模型 — 輸入 `/model`，用方向鍵選擇 Haiku
  4. 下一個簡單指令測試 — 「用一句話解釋 AI Agent 是什麼」
  5. 感受速度差異後，用 `/model` 換回 Sonnet
  [/flow]
  ```
