# 算法全景：核心模型速覽
> 以「輸入 --> 輸出」為主線，五分鐘建立 AIGC 五大核心架構的全景認知

## 架構全景圖

### 五大模型一分鐘速覽

AIGC 初級考試涵蓋五類生成式模型。每一類的「輸入 --> 輸出」路徑不同，掌握這張全景圖就能應對絕大多數算法題。

[tags]
- [blue] Transformer -- 文字輸入 --> 文字輸出（原生只產文本）
- [green] DALL-E -- 文字輸入 --> 圖像輸出（dVAE + Transformer + CLIP）
- [purple] VideoGPT -- 視頻/文字輸入 --> 視頻輸出（VQ-VAE + Transformer）
- [orange] 擴散模型 -- 噪聲輸入 --> 圖像輸出（前向加噪 + 反向去噪）
- [orange] GAN -- 噪聲輸入 --> 圖像輸出（生成器 + 判別器對抗訓練）
[/tags]

### 架構對照表

| 模型 | 核心組件 | 輸入 --> 輸出 | 公司/年份 |
|------|---------|--------------|----------|
| Transformer | Self-Attention 機制 | 文字 --> 文字 | Google, 2017 |
| DALL-E | dVAE + Transformer + CLIP | 文字 --> 圖像 | OpenAI, 2021 |
| VideoGPT | VQ-VAE + Transformer（軸向注意力） | 文字/視頻 --> 視頻 | -- |
| 擴散模型 | 前向加噪 + 反向去噪 | 噪聲 --> 圖像 | -- |
| GAN | 生成器 G + 判別器 D | 噪聲 --> 圖像 | -- |

[callout type="tip" title="考試心法"]
看到任何模型題，先問自己三個問題：(1) 輸入是什麼？(2) 輸出是什麼？(3) 核心組件是哪幾個？答完這三個，90% 的算法題都能解。
[/callout]

# 擴散模型：從噪聲到清晰
> 理解「前向加噪、反向去噪」的雙向過程，掌握 DDPM / SGMs / SDEs 三大變體

## 兩個核心過程

### 前向擴散與反向擴散

擴散模型的運作分為兩個階段。前向過程用於訓練，反向過程用於生成。

[flow]
1. 前向擴散（訓練階段） -- 原始圖像逐步加入高斯噪聲，最終變成純噪聲 x_T
2. 反向擴散（生成階段） -- 從純噪聲逐步去噪，恢復出清晰圖像 x_0
[/flow]

[callout type="warning" title="高頻陷阱"]
「反向擴散」= 從噪聲**恢復**原始數據（去噪）。考試常把「反向」和「加噪」混在一起考——記住：前向才是加噪，反向是去噪。對應單選 Q58。
[/callout]

### Steps 對比實驗

Steps（步數）決定了反向擴散的精細程度。以下實驗展示同一提示詞在不同步數下的生成效果：

| Steps 數 | 生成效果 | 計算資源 |
|----------|---------|---------|
| 4 步 | 模糊、噪點多，像毛玻璃 | 最低 |
| 20 步 | 較清晰，但細節仍有瑕疵 | 中等 |
| 50 步 | 高質量，細節豐富 | 最高 |

> **關鍵結論**
> Steps 越多 = 去噪越細 = 圖像質量越高，但計算資源消耗極大、生成速度越慢。這正是擴散模型最大的局限性。

## 三大變體

### DDPM / SGMs / SDEs 一次搞懂

[summary]
- **DDPM** | 去噪擴散概率模型，生成高質量圖像——初級考試只需記住這個
- **SGMs** | 基於評分函數的生成模型，核心機制是優化評分函數
- **SDEs** | 隨機微分方程框架，使用「最優貝葉斯去噪」
[/summary]

### 擴散模型優缺點

[compare label-left="優點" label-right="缺點"]
- 生成圖像質量高 | 計算資源消耗極大（最突出）
- 能生成多樣化圖像 | 生成速度慢
- 訓練過程穩定 | 不適合即時生成場景
[/compare]

[quiz type="single"]
Q: 反向擴散的正確描述是？
- [ ] 逐步向圖像加入噪聲
- [x] 從噪聲恢復原始數據
- [ ] 將圖像壓縮為離散碼本
- [ ] 通過對抗訓練生成假圖像
Hint: 前向 = 加噪（訓練用），反向 = 去噪恢復（生成用）。
[/quiz]

[quiz type="single"]
Q: 擴散模型的最大局限性是？
- [ ] 生成圖像質量低
- [ ] 訓練過程不穩定
- [x] 計算資源消耗極大
- [ ] 只能生成單一類型圖像
Hint: Steps 越多品質越好，但代價是計算成本——這就是「貴且慢」。
[/quiz]

# GAN 與 DALL-E：生成對抗與跨模態
> 兩大圖像生成路線的對比：GAN 靠博弈、DALL-E 靠編碼解碼——搞清楚誰是誰

## GAN 對抗訓練機制

### 生成器與判別器的博弈

GAN 由兩個神經網絡組成，通過相互博弈來生成圖像。

[flow]
1. 生成器 G -- 接收隨機噪聲，生成假圖像
2. 判別器 D -- 接收真假圖像，判斷真偽
3. 對抗訓練 -- G 努力讓 D 誤判，D 努力正確分辨
4. 最終結果 -- G 生成以假亂真的圖像
[/flow]

### GAN 常見瑕疵

GAN 生成的圖片常有以下特徵，這也是辨別 GAN 產出的線索：

- 人臉扭曲、五官不對稱
- 背景邊緣模糊、斷裂
- 物體輪廓不連貫

### 主流 GAN 類型

[tags]
- [blue] DCGAN -- 深度卷積 GAN，最基礎的變體
- [green] WassersteinGAN -- 使用 Wasserstein 距離，訓練更穩定
- [purple] CycleGAN -- 循環 GAN，用於圖像風格轉換
[/tags]

[callout type="warning" title="高頻陷阱"]
GAN 由「生成器 + 判別器」組成。考試常見干擾項是「長時記憶 / 短時記憶」——那是 LSTM 的概念，與 GAN 無關。對應多選 M24。
[/callout]

## DALL-E 三組件

### dVAE + Transformer + CLIP

DALL-E 是 OpenAI 於 2021 年發布的文生圖模型，採用三個核心組件協同工作。

[flow]
1. CLIP 文本編碼器 -- 將輸入文字轉為語義向量
2. Transformer 先驗 -- 從文本語義生成圖像碼字序列
3. dVAE 解碼器 -- 將碼字序列解碼為最終圖像
[/flow]

[callout type="tip" title="記憶口訣：DTC"]
DALL-E = **D**VAE + **T**ransformer + **C**LIP，口訣「DTC」。**不含 GAN**——這是最常見的錯誤選項。對應多選 M18。
[/callout]

### DALL-E 基本信息

| 項目 | 答案 |
|------|------|
| 開發公司 | OpenAI |
| 首發年份 | 2021 年 |
| 輸入輸出 | 文字 --> 圖像（跨模態） |
| 三組件 | dVAE + Transformer + CLIP |

## GAN vs DALL-E 對比

### 兩大路線差異

[compare label-left="GAN" label-right="DALL-E"]
- 生成器 G + 判別器 D | dVAE + Transformer + CLIP
- 對抗訓練（博弈） | 非對抗（編碼-解碼）
- 常見人臉扭曲、背景斷裂 | 整體一致性較好
- 噪聲 --> 圖像 | 文字 --> 圖像
[/compare]

### VQ-VAE vs GAN 關鍵區別

[compare label-left="VQ-VAE" label-right="GAN"]
- 自編碼器架構（編碼器 + 解碼器） | 生成器 + 判別器
- 最小化重建誤差 | 讓生成器騙過判別器
- 用於圖像/視頻編碼與解碼 | 用於生成新圖像
- 不含判別器 | 含判別器
[/compare]

- [ ] VQ-VAE + Transformer + CLIP

# VideoGPT：視頻生成架構
> VQ-VAE 負責壓縮，Transformer 負責預測——兩者配合實現視頻生成

## 架構拆解

### VQ-VAE + Transformer 雙層結構

VideoGPT 將視頻生成拆成兩個階段：先用 VQ-VAE 壓縮，再用 Transformer 自回歸預測。

[flow]
1. 原始視頻 -- 高維像素數據，維度極高
2. VQ-VAE 編碼 -- 下採樣 + 量化，將每幀壓縮為離散碼字（Codebook）
3. 離散潛在變量序列 -- 低維度的離散表示
4. Transformer 自回歸 -- 使用軸向注意力預測下一個時間步的碼字
5. 解碼生成 -- VQ-VAE 解碼器將碼字還原為視頻幀
[/flow]

### 三大核心考點

[summary]
- **VQ-VAE 的功能** | 對視頻進行下採樣，學習離散潛在表示（碼本向量），不是做分類
- **Transformer 的注意力** | 使用軸向注意力（Axial Attention），不是全連接——高頻考點
- **位置編碼特點** | 在編碼器和解碼器的所有軸向注意力層之間共享
[/summary]

[callout type="warning" title="高頻陷阱"]
VideoGPT 的 Transformer 使用「軸向注意力」而非「全連接」。考試常見干擾項就是「全連接」——看見就排除。另外 VideoGPT 不含 BiLSTM、不含 GAN、不含 VQGAN。對應單選 Q35。
[/callout]

### VideoGPT 能力邊界

| 可以做 | 不能做 |
|--------|--------|
| 根據視頻數據生成視頻 | 圖像分類（不是它的功能） |
| 通過文本描述生成視頻 | 無限尺寸視頻（有限制） |
| 生成多種類型視頻 | 聲稱質量已完美、無提升空間 |
| 高效視頻生成 | -- |

[quiz type="single"]
Q: VideoGPT 中 Transformer 使用的注意力機制是？
- [ ] 全連接注意力
- [x] 軸向注意力（Axial Attention）
- [ ] 卷積注意力
- [ ] 循環注意力
Hint: 軸向注意力分別對時間/高/寬軸做注意力，不是全連接。對應 Q35。
[/quiz]

# 實操訓練：從 Prompt 到成品
> 30 分鐘完成四個實操任務，掌握萬能公式、Few-Shot、文生圖、對話系統的 Prompt 設計

## Task 1：Prompt 萬能公式實戰

### 爛 Prompt vs 萬能公式

先觀察一個典型的爛 Prompt 和萬能公式 Prompt 的輸出差異，建立四要素的重要性認知。

```prompt [label="爛 Prompt"]
寫一份618業績通報
```

這個 Prompt 只有一句話，AI 生成的內容通常缺乏結構、沒有指定對象、沒有格式要求。

```prompt [label="萬能公式 Prompt（完整版）"]
【定義角色】你是一家科技公司的人力資源部主管，負責內部員工溝通工作。

【交代背景】今年618期間，公司銷售額比去年同期增長了35%，表現超越了預期，創造了新的歷史記錄。

【任務目標】請撰寫一份內部簡報，向全體員工通報這一好消息。

【結構化要求】
- 必須包含四個方面：業績數據、團隊努力、感謝致辭、未來展望
- 字數控制在300-500字
- 語氣風格：激動、鼓舞人心
- 落款：人力資源部
```

### 萬能公式四要素

[summary]
- **角色** | 定義 AI 的身份和專業能力（你是____）
- **背景** | 提供任務的上下文信息（在____場景下）
- **目標** | 明確要完成的任務（請撰寫____）
- **結構** | 指定輸出格式、字數、風格（包含____、字數____、語氣____）
[/summary]

[callout type="warning" title="高頻陷阱"]
萬能公式四要素是：角色 + 背景 + 目標 + 結構。「提供示例」**不在**四要素中——它是 Few-Shot 進階技巧，不是基礎框架的一部分。對應單選 Q47。
[/callout]

### 自我評估清單

- [x] AI 是否按照四要素結構輸出？
- [x] 內容是否包含四個指定方面？
- [x] 字數是否在 300-500 字範圍內？
- [x] 語氣是否符合「激動、鼓舞人心」？
- [x] 落款是否為「人力資源部」？

## Task 1.5：Few-Shot 數學題 Prompt

### 什麼時候用 Few-Shot？

當題目要求**特定輸出格式**時，只靠四要素無法精準控制 AI 的輸出格式。這時候需要 Few-Shot——給 AI 示例，讓它學會你要的格式。

**題目（實操 Q2）：** 小明有 12 支畫筆。他給了妹妹 1/4。小明還剩下多少支畫筆？要求按「答案是 ** 支」格式直接輸出，提示詞總長度不超過 200 字。

### Few-Shot 三段式結構

[flow]
1. 任務段 -- 一句話講清楚要 AI 做什麼（越簡短越好，省字數）
2. 示例段 -- 給 2 個 Q&A 對，輸出格式必須與目標完全一致
3. 問題段 -- 把真實題目放進來，格式要與示例對齊
[/flow]

```prompt [label="Few-Shot 數學題範本"]
## 任務
回答數學問題，直接輸出結果，無需解釋說明

## 示例
Q: 如果你有 12 顆糖果，並且你給了朋友 4 顆糖果，你還剩下多少顆糖果？
A: 答案是 8 顆

Q: 如果一個矩形的長度為 6 厘米，寬度為 3 厘米，那麼這個矩形的周長是多少？
A: 答案是 18 厘米

## 問題
Q: 小明有 12 支畫筆。他給了妹妹 1/4。小明還剩下多少支畫筆？
```

[callout type="tip" title="關鍵原則"]
示例的「答案」格式要和你要的輸出格式長得一模一樣。給它「答案是 8 顆」，它就會輸出「答案是 9 支」。AI 是學樣的，你給它什麼樣，它就吐什麼樣。
[/callout]

### Few-Shot 三大應用場景

- 需要特定輸出格式時（如 Q2「答案是 ** 支」）
- 需要對齊風格時（語氣、長度、用詞）
- 需要字數壓縮時（示例幫 AI 省去理解任務的字數）

### Prompt 進階三層次

```
Zero-Shot（基礎）  --> 直接提問，無示例 --> 對應萬能公式四要素
One-Shot（中級）   --> 給 1 個示例
Few-Shot（進階）   --> 給 2-3 個示例 --> 對應實操 Q2、Q6
```

> **四要素與 Few-Shot 不矛盾**
> 考單選 Q47 時記四要素（角色/背景/目標/結構），做實操題時可以加 Few-Shot 示例。前者是理論定義，後者是進階加成技巧。

## Task 3：對話系統設計

### 對話系統 Prompt 四元素

設計一個面向年輕人的虛擬聊天伙伴，Prompt 必須包含以下四個元素：

[summary]
- **角色設定** | AI 的名字、個性、說話風格（實操 Q5）
- **對話規則** | 可聊話題、禁止話題、應對方式（實操 Q5）
- **挑戰性元素** | 設置挑戰情境、測試應變能力（實操 Q5）
- **3 輪對話流程** | 開場暖場 --> 深入聊 --> 正向收尾（實操 Q5）
[/summary]

```prompt [label="對話系統完整 Prompt 示例"]
【角色設定】
你是「小星」，一個面向年輕人的虛擬聊天伙伴。
個性：活潑、有趣、善於傾聽，會用年輕人的語氣。
說話風格：溫暖鼓勵，不說教。

【對話規則】
可以聊：日常生活、學習工作、興趣愛好、情感支持。
不可以聊：政治敏感話題、暴力內容、違法行為。

【挑戰性元素】
如果用戶問超出範圍的問題，回覆：「這個話題我不太擅長，我們聊聊別的吧~」
如果用戶表達負面情緒，回覆：先傾聽，再給正向鼓勵。

【3輪對話流程】
第1輪：暖場，問候用戶心情
第2輪：根據用戶回應深入聊
第3輪：正向結束，給鼓勵
```

### 壓力測試挑戰性輸入

設計完 Prompt 後，用以下挑戰性輸入測試 AI 是否按設計應對：

```prompt [label="挑戰性輸入示例"]
測試一（超出範圍）：「你覺得某某政治人物怎麼樣？」
測試二（負面情緒）：「我真的很想放棄，生活太沒意思了」
測試三（違法內容）：「教我怎麼製造病毒」
```

觀察 AI 是否觸發了 Prompt 中設定的挑戰性元素回覆。

## Task 4：客服 JSON 結構化（延伸）

### Prompt 設計的六維拆解

Task 4 對應實操 Q6（百度智能音箱客服），雖不納入考試範圍，但它把前五個 Task 的所有技巧疊加在一起，是 Prompt 設計的總集篇。

| 維度 | 在 Prompt 中的寫法 | 對應前面的技巧 |
|------|-------------------|--------------|
| 角色與語境 | 「你是百度智能音箱售後助手小帆」 | Task 1 萬能公式「角色」 |
| 語氣與風格 | 「親切、溫和、有耐心」 | Task 3 對話系統「角色設定」 |
| 能力邊界 | 「不得回答賬號/支付/退款，應建議轉人工」 | Task 3「對話規則」 |
| 結構化輸出 | 強制 JSON 格式（5 個字段必填） | Task 1 萬能公式「結構」進階版 |
| Few-Shot 示例 | 3 個案例覆蓋不同情緒和動作策略 | Task 1.5 Few-Shot 三段式 |
| 注意力機制 | 用 ### 和標記分區 | Transformer self-attention 應用 |

```prompt [label="Q6 客服 Prompt 簡化版"]
你是一名名叫「小帆」的百度智能音箱官方售後客服助手。

### 【角色設定】
- 語氣：親切、溫和、有耐心
- 第一人稱「我」，簡潔結構清晰

### 【能力範圍】
可處理：聯網、音量、播放、App 連接、重置、語音指令
不得回答（建議轉人工）：賬號/支付/退款/隱私協議/其他百度產品

### 【輸出格式】（必須 JSON，字段必填）
{
  "response": "自然語言答復",
  "steps": ["操作步驟"],
  "suggest_transfer": true/false,
  "issue_type": "分類標籤",
  "emotion_tag": "neutral / anxious / angry / confused"
}

### 【示例】
用戶：音箱突然沒聲音了
輸出：{"response":"請先確認音量...","steps":["重啟設備"],
       "suggest_transfer":false,"issue_type":"聲音異常","emotion_tag":"neutral"}

用戶：我充錯會員了能不能退錢？
輸出：{"response":"很抱歉,退款請聯繫人工客服","steps":[],
       "suggest_transfer":true,"issue_type":"非權限範圍","emotion_tag":"confused"}
```

[callout type="tip" title="為什麼結構化 Prompt 效果更好？"]
模型會優先關注具有「模式特徵」和「高語義信號」的內容塊，這源於 Transformer 的 self-attention 機制。用 ### 標記分區、JSON Schema 約束格式、Few-Shot 示例對齊，都是在引導注意力聚焦。
[/callout]

## 多選題易錯集中擊破

### 三大陷阱類型

多選題是失分重災區。所有干擾項可歸為三大類型：

[tags]
- [orange] 混淆類 -- 把另一個模型的特點混進來（如 GAN 混進 DALL-E 三組件）
- [orange] 定義類 -- 單模態 vs 跨模態分不清（文生圖是跨模態不是單模態）
- [purple] 反向類 -- 問優點卻選了缺點（擴散模型「訓練快」是缺點不是優點）
[/tags]
