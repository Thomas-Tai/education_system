# 基礎入門：認識 Fusion 的節點世界
> Fusion 是 DaVinci Resolve 內建的節點式合成引擎——理解節點邏輯，是掌握所有特效合成的起點

## Fusion 頁面概覽

### Fusion 與 Edit 頁面的差異
- **Edit 頁面**：時間軸驅動，以片段順序為核心的線性剪輯
- **Fusion 頁面**：節點驅動，以畫面合成為核心的非線性處理
- Fusion 的每一次效果都是一個節點，節點之間的連線決定資料流向
- 兩個頁面共享同一個時間軸，可在之間自由切換

### 介面導覽

[flow]
1. 節點區（Node Editor）— 建立與連接節點的核心工作區
2. 檢視器（Viewer）— 即時預覽合成結果，支援雙視窗比對
3. 檢查器（Inspector）— 調整選取節點的參數
4. 時間軸（Timeline）— 控制關鍵影格與動畫時間
[/flow]

- 熟悉快捷鍵可大幅提升工作效率
- 雙視窗模式：左邊顯示原始畫面，右邊顯示合成結果

## 節點基礎

### 什麼是節點？
- 節點是 Fusion 的最小處理單元，每個節點執行一項特定任務
- 資料從左到右流動：輸入 → 處理 → 輸出
- 節點之間透過連線傳遞影像資料

[compare label-left="圖層式（After Effects）" label-right="節點式（Fusion）"]
- 上下堆疊，上層覆蓋下層 | 左右流向，輸入連接到輸出
- 圖層多時難以追蹤依賴關係 | 每條連線清楚標示資料來源
- 修改中間層可能影響所有上層 | 修改單一節點不影響其他分支
- 適合直覺式、線性合成 | 適合複雜、多分支的非線性合成
[/compare]

[dl]
- Merge | 將前景與背景合成的節點，黃端接前景、綠端接背景
- Delta Keyer | 高品質摳像工具，內建溢色抑制
- Tracker | 追蹤畫面中物體的運動軌跡，可套用至遮罩或特效
- pEmitter | 粒子發射源，控制粒子的出生位置、方向與數量
- Shape 3D | 將 2D 影像轉換為 3D 空間中的平面物件
- Macro | 將多個節點打包為單一節點的巨集，方便複用
[/dl]

[tags]
- [blue] MediaIn — 匯入媒體素材
- [green] Effect — 套用視覺效果
- [purple] Merge — 合成多個影像來源
- [orange] MediaOut — 輸出最終結果
[/tags]

### 節點操作技巧
- 拖曳節點輸出端到另一節點輸入端，建立連線
- 斷開連線：選取連線後按 Delete
- **Branch**：從同一節點拉出多條連線，建立分支處理
- **Insert**：在兩個已連接的節點之間插入新節點

[callout tip title="節點連接注意事項"]
- 節點的三角形方向：大三角為輸出端（右側），小三角為輸入端（左側）
- 不同顏色的輸入端代表不同資料類型：藍色為 2D 影像、綠色為背景、黃色為前景
- 連線時若類型不匹配，該連線不會生效
- 使用 Branch 時，建議為每個分支命名（Label），方便後續追蹤
[/callout]

[callout tip title="節點思維 vs 圖層思維"]
After Effects 採用圖層堆疊，上層覆蓋下層；Fusion 採用節點流向，資料沿連線傳遞。節點架構在處理複雜合成時更具彈性，每個處理步驟獨立可見、可調整，不用擔心圖層順序造成的意外覆蓋。
[/callout]

### 常用快捷鍵

```prompt [label="節點操作"]
空白鍵 — 搜尋並新增節點
Ctrl+T — 新增 Transform 節點
Ctrl+D — 新增 Merge 節點
Ctrl+Shift+D — 斷開選取節點的連線
1 / 2 — 切換左/右檢視器顯示
```

# 2D 合成：遮罩、摳像與圖層合成
> 遮罩與摳像是 Fusion 合成的兩大支柱——精確控制畫面中哪些區域受影響，是所有特效的基礎

## 遮罩（Mask）

### 遮罩的基本概念
- 遮罩定義影像中「受影響」與「不受影響」的區域
- 白色區域 = 作用範圍，黑色區域 = 不作用
- 遮罩可以套用在任何節點上，控制效果的局部範圍

### 遮罩類型與應用

[flow]
1. Polygon Mask — 多邊形遮罩，適合幾何形狀的精確選取
2. Bitmap Mask — 點陣圖遮罩，從外部圖片產生遮罩形狀
3. Path Mask — 路徑遮罩，適合需要動態追蹤的曲線選取
[/flow]

- 遮罩可加入柔邊（Soft Edge）讓邊緣更自然
- 多個遮罩可透過 Boolean 運算組合：Union、Subtract、Intersect

### 動態遮罩追蹤
- 使用 Tracker 節點追蹤畫面中的運動軌跡
- 將追蹤資料套用到遮罩，讓遮罩跟隨目標移動
- 支援單點追蹤與多點追蹤
- 追蹤結果可套用到位移、旋轉、縮放等變換

## 摳像（Keying）

### Delta Keyer
- DaVinci Resolve 內建的高品質摳像工具
- 直接在畫面上選取要移除的顏色範圍
- 自動處理邊緣溢色（Spill Suppression），無需額外節點

[flow]
1. 選取背景色 — 在檢視器中點選要移除的顏色
2. 調整摳像範圍 — 透過 Matte 檢視模式微調邊緣
3. 溢色抑制 — 處理主體邊緣的背景色殘留
4. 邊緣調整 — 羽化與收縮讓摳像邊緣更自然
[/flow]

### 更精細的摳像：Erase/Restore
- 在 Delta Keyer 內使用 Erase/Restore 筆刷
- **Erase**：手動移除殘留的背景區域
- **Restore**：手動還原被誤移除的主體區域
- 適合處理頭髮、半透明材質等摳像難點

[callout warning title="摳像的關鍵在拍攝端"]
再好的摳像工具也救不回拍攝品質太差的素材。均勻打光的綠幕/藍幕、主體與背景足夠的距離、避免壓縮失真——這些前置作業決定了摳像的成敗。
[/callout]

## Merge 合成

### Merge 節點的連接邏輯
- **Foreground（前景）**：連接到 Merge 的黃色輸入端
- **Background（背景）**：連接到 Merge 的綠色輸入端
- 合成結果 = 前景疊在背景之上

### 常用混合模式

[reveal title="Merge 前後接反會怎樣？"]
如果前景接在綠色端、背景接在黃色端，合成結果會相反——原本該在上面的畫面跑到下面，原本該當背景的變成前景。這是新手最常犯的錯誤之一，畫面全黑或效果不對時，第一件事就是檢查 Merge 的連線方向。
[/reveal]
- **Normal**：標準前景覆蓋背景，受 Alpha 通道控制
- **Screen**：適合光暈、光芒等發光效果合成
- **Multiply**：適合陰影、暗部疊加
- **Overlay**：保留背景明暗層次的同時疊加紋理

- [x] 否（白色 = 作用範圍，黑色 = 不作用）

# 文字與動態圖像：Title 與動畫
> 文字不只是一行訊息——透過動態圖像，文字本身就是視覺焦點

## 文字節點

### Text+ 節點
- Fusion 專用的向量文字節點
- 支援字型、大小、間距、行距等完整排版控制
- 可直接在檢視器中拖曳定位文字
- 支援多行文字與自動換行

### 文字動畫技巧
- 在 Inspector 中對參數按右鍵 → Animate，加入關鍵影格
- 常見動畫：淡入淡出、位移滑入、縮放彈跳
- 使用 Spline 面板調整動畫曲線，控制加速與減速
- 使用 Modifiers（如 Wiggle、Noise）建立隨機動態效果

## 動態圖像實戰

### 常見排版版型

[flow]
1. Lower Third — 人名與職稱的下半部標題動畫
2. Callout — 箭頭指向搭配說明文字的標注動畫
3. Credit Roll — 片尾工作人員名單的滾動動畫
[/flow]

- 善用 Merge 節點將文字與裝飾元素（線條、色塊）組合
- 統一動畫節奏：進場、停留、退場各階段保持一致的速度感
- 建立可複用的文字模板，提升專案效率

[callout tip title="動態圖像的核心是節奏"]
好的動態圖像不是特效越多越好，而是每個元素在正確的時間點出現與消失。觀察專業作品的進場節奏——簡潔、有層次、不搶畫面。
[/callout]

# 粒子與特效：動態視覺效果
> 粒子系統讓靜態畫面活了過來——從飄落的雪花到爆裂的火花，一切由參數驅動

## 粒子系統

### pEmitter 與粒子流程
- **pEmitter**：粒子的發射源，控制出生位置、方向與數量
- 粒子流程本質上也是節點流程：pEmitter → pTurbulence → pRenderer
- pRenderer 將粒子渲染為畫面上的視覺元素

### 粒子控制參數

[flow]
1. pEmitter — 數量、速度、生命週期、大小
2. pTurbulence — 亂流場，讓粒子產生有機的隨機運動
3. pDirectionalForce — 方向力（如重力、風力）
4. pRender — 渲染模式（點、線、精靈圖）
[/flow]

- 粒子數量直接影響渲染效能，寧可少而精也不要過量
- 生命週期（Lifespan）控制粒子從出生到消失的時間
- 結合隨機參數讓粒子行為更自然，避免機械感

### 粒子實戰：飄雪效果
- 建立 pEmitter，設定由上往下的發射方向
- 加入 pTurbulence 產生左右飄移
- 加入 pDirectionalForce 模擬微弱重力
- 調整粒子大小與透明度變化，讓遠近有別

## 常用特效節點

### 光暈與發光
- **Glow**：為亮部區域添加柔光擴散效果
- **Blur**：模糊處理，可搭配遮罩做局部柔焦
- **Light Wrap**：將背景光線「包裹」到前景邊緣，增強合成真實感

### 色彩調整
- **BrightnessContrast**：調整整體明暗對比
- **ColorCurves**：精細控制色彩曲線
- **HueSaturation**：調整色相與飽和度

[callout tip title="特效是為敘事服務的"]
特效的目的是強化故事，而非取代故事。觀眾不會因為粒子多而感動，但會因為粒子出現的時機恰到好處而驚嘆。每一次加特效前，先問自己：這個效果對畫面說了什麼？
[/callout]

# 3D 空間：攝影機、燈光與深度合成
> Fusion 的 3D 空間讓合成不再局限於平面——攝影機運動、燈光交互與深度合成，打造真正的三維視覺體驗

## 3D 合成基礎

### 從 2D 邁向 3D

[tabs]
[tab label="2D 節點"]
- 在平面空間（X, Y）中處理合成
- 使用 Merge 節點疊加前景與背景
- 適合遮罩、摳像、文字動畫等平面效果
- 不需要 Camera 3D 或 Renderer 3D
[/tab]
[tab label="3D 節點"]
- 在立體空間（X, Y, Z）中處理合成
- 使用 Merge 3D 將所有物件合併到同一場景
- 需要 Shape 3D 將 2D 素材轉為 3D 物件
- 必須有 Camera 3D 決定視角、Renderer 3D 渲染輸出
[/tab]
[/tabs]
- 將 2D 素材透過 Shape 3D 節點轉為 3D 平面
- 3D 空間中可自由旋轉、位移、縮放每個元素
- 攝影機決定觀看視角，燈光決定明暗層次

### 3D 場景組成

[flow]
1. Shape 3D — 將 2D 影像放入三維空間
2. Camera 3D — 定義觀看視角與透視
3. Light 節點 — 建立場景照明（方向光、點光源、聚光燈）
4. Merge 3D — 將所有 3D 元素合併為同一場景
5. Renderer 3D — 將 3D 場景渲染為 2D 輸出
[/flow]

## 攝影機與燈光

### Camera 3D
- 控制視角：焦距（Focal Length）、視野（FOV）、景深（Depth of Field）
- 攝影機可加入關鍵影格動畫，模擬運鏡效果
- 善用攝影機動畫讓靜態素材產生動態感

### 燈光類型
- **Directional Light**：平行光，模擬遠處光源如太陽
- **Point Light**：點光源，從單一位置向四面八方照射
- **Spot Light**：聚光燈，有方向與範圍的錐形光
- 燈光強度、顏色與陰影設定會直接影響合成真實度

### 深度合成技巧
- 利用 3D 空間的 Z 軸深度排列元素
- 霧化（Fog）效果強化遠近層次感
- 景深模糊讓焦點外的元素自然虛化

[reveal title="如何用最少燈光做出真實感？"]
一盞主光（Key Light）決定主要明暗與陰影方向，一盞補光（Fill Light）以較低強度柔化暗部，再加一盞背光（Rim Light）從後方打出輪廓邊緣。三盞燈的經典配置比五盞燈全開更容易控制、更省效能，而且畫面更有層次。
[/reveal]

# 實戰整合：進階技巧與工作流程
> 將前面學到的所有技巧整合——從素材準備到最終輸出，建立可複用的專業工作流程

## 模板與巨集

### 建立可複用的巨集（Macro）
- 選取多個節點 → 右鍵 → Create Macro
- 巨集將多個節點打包為單一節點，簡化流程
- 可自訂巨集的公開參數，只暴露需要調整的控制項
- 巨集儲存後可在其他專案重複使用

### 從 Fusion 到 Edit
- Fusion 合成結果自動回傳到 Edit 頁面的時間軸
- 在 Edit 頁面可繼續調整片段長度、轉場與音訊
- 雙向連動：Edit 頁面的修改會即時反映到 Fusion

## 效能與除錯

### 效能優化技巧

[callout warning title="效能優化注意事項"]
- 粒子數量直接影響渲染效能，寧可少而精也不要過量
- 3D 場景中的 Renderer 3D 是最耗時的節點之一，預覽時可降低抗鋸齒等級
- 大量 Blur 節點會嚴重拖慢速度，可考慮先用 Fast Blur 替代
- 完成預覽後再開啟 Render Cache 做最終輸出
[/callout]
- 減少不必要的節點數量，合併可合併的處理步驟
- 使用 Proxy 模式降低預覽解析度，加快即時預覽速度
- 善用 Cache 機制：Render Cache 設為 Auto 或 User 定義
- 關閉不需即時預覽的分支，使用 Disable 節點暫時停用

### 常見問題排除

[flow]
1. 畫面全黑 — 檢查 Merge 的前背景是否接反
2. 效果沒作用 — 確認遮罩方向（白色為作用區）
3. 邊緣鋸齒 — 增加遮罩柔邊或開啟抗鋸齒
4. 渲染過慢 — 檢查 Proxy 設定與 Cache 狀態
[/flow]

[accordion]
[item title="如何把做好的特效存成模板？" open]
選取多個節點後按右鍵，選擇 Create Macro。在彈出視窗中勾選你希望暴露給使用者調整的參數，儲存到 Fusion 的 Templates 目錄即可。之後在任何專案中都能從 Effects Library 拖曳使用。
[/item]
[item title="為什麼我的合成畫面是半透明的？"]
最常見的原因是 Merge 節點的 Alpha 通道設定。檢查前景是否有透明的 Alpha，或是否誤用了 Screen/Multiply 等混合模式。切換回 Normal 模式確認基礎合成效果。
[/item]
[item title="粒子特效在預覽時很卡怎麼辦？"]
先在 pEmitter 中將 Quantity 降到最低可做預覽的數值，確認運動方向與生命周期設定無誤後再逐步調高。也可以使用 Proxy 模式降低預覽解析度。
[/item]
[/accordion]
