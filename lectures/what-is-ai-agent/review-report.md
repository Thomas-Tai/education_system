## 各章節改善建議

### 章節一：破冰：從 ChatGPT 談起

**現況**：
- 說明層 — 5 張卡片清楚定義「傳統 AI」與「AI Agent」，類比（管家 vs 翻譯機）對初學者友善
- 範例層 — `prompt block` 展示 AI Agent 定義，`vote` 作為開場互動很到位
- 實作層 — 偏弱。本章是觀念鋪陳，沒有動手指引

**建議**：

- 實作層：在「什麼是 AI Agent？一句話定義」子章節後，加入一張「動手記錄」卡片，讓學員在 30 秒內寫下自己腦中對 AI Agent 的想像，作為後續章節的對照。

  ```markdown
  ### 30 秒小練習：寫下你的定義
  - 在筆記本或手機寫一句話：「AI Agent 是 ______」
  - 不用精準，寫直覺就好
  - 課程最後會請你回頭比對，看看認知變化
  ```

### 章節二：傳統 AI vs AI Agent：完整對比

**現況**：
- 說明層 — 三個 compare 元件把差異講得非常清楚，八維度對比是本課亮點
- 範例層 — `prompt block` 展示傳統 AI vs Agent 的對話差異，`quiz` 收尾驗收觀念
- 實作層 — 完全缺失。學員只能「看」對比，沒有「感受」對比的機會

**建議**：

- 實作層：在「看出差異了嗎？」子章節後，加入一個「對照體驗」的 checklist，引導學員實際拿 ChatGPT 跑一次實驗。

  ```markdown
  ### 對照體驗：同一題問兩次
  - [ ] 打開 ChatGPT，輸入「幫我規劃五天日本行程」，記錄它的回應
  - [ ] 再追問「這家飯店有空房嗎？」觀察它是否承認無法查詢
  - [ ] 回來對照本課的八維度表，標記哪些維度在 ChatGPT 身上能看到
  - [ ] 思考：如果有一個 Agent 能做到，它需要哪些元件？（下一題伏筆）
  ```

### 章節八：真實案例：從理論到實踐

**現況**：5 個案例（旅遊 / 客服 / 開發 / 研究 / 知識管理）全部成功，四大元件在每個案例中都清楚標註。

**建議**：

- 案例都是成功案例。可在結尾加一個「失敗案例警示」卡片，平衡視角：

  ```markdown
  ### 失敗案例：什麼時候不該用 Agent？

  - 容錯率極低（醫療診斷、金融交易） — 目前仍需人類主導
  - 任務定義不清 — Agent 會陷入無限反思循環
  ```

## 全篇品質

**裸段落**：無。所有段落都在卡片、引言、子章節內，結構嚴謹。

**卡片密度**：
- 9 章共 36 張卡片，平均每章 4 張，密度健康
- 最高密度：第 5 章（規劃）10 張 — 接近 7–10 張的「偏多」範圍，但因拆成 3 個子章節（拆解 / 反思 / 動手），仍可消化
- 最低密度：第 3 章（核心架構）2 張 — 作為總覽章節合理，不需擴充

**元件多樣性**：
- flow 共 14 個 — 豐富
- prompt block 共 8 個 — 充足
- insight box 共 8 個 — 每章 1 個，節奏良好
- compare 共 8 個 — 對比式教學是本課特色
- image-text 共 5 個 — 每章平均不到 1 個，分布合理
- quiz 共 5 個 — 每章 1 個，驗收節奏良好
- vote 1 個（開場）— 建議第 6 或 7 章可再加 1 個（例如「你最在意哪一種記憶？」）
- bonus 1 個（第 5 章）— 建議第 7 章可再加 1 個補充（例如 MCP 協議細節）
- summary 1 個（總結）— 合規
- tags 共 6 個 — 分布均勻

**圖片多樣性**：
- chapter hero：4 張（第 4 / 5 / 6 / 7 章），符合 2–4 張建議
- 內文獨立圖片：5 張，分布均勻
- image-text：5 個，每章 1 個左右，不造成視覺單調
- 行內圖片：0 個（正常）
- TODO 殘留：0 個（全部 `![alt](images/...)` 都是有意義路徑）
- 圖片 alt：全部有意義描述，無空泛 `image` / `img`

**時間估算**：預估總時長 **224–279 分鐘**（約 3.7–4.6 小時），目標 4 小時（240 分鐘），時間充裕。若講述速度中等，可剛好壓在 4 小時完成。

## 總結評語

這份 content.md 整體品質很高：

- **結構成熟**：9 章邏輯遞進清晰，從破冰 → 對比 → 架構 → 四大元件 → 案例 → 總結，閱讀動線流暢。
- **教學節奏佳**：四大元件章節（第 4–7 章）都完整涵蓋「說明 → 範例 → 實作」三層次，並搭配 quiz 驗收。
- **初學者友善**：大量生活類比（管家、旅行、大象放冰箱），術語首次出現都有解釋。
- **元件使用精準**：compare、flow、prompt block、image-text 的搭配展現專業教學設計。

**主要弱點**：開場前兩章（破冰、對比）實作層偏弱，建議依「優先修改清單」前兩項補強，即可達 Phase 3 build 標準。
